<task type="auto">
  <name>Task 2: Create auth interceptor and rate limit interceptor platform packages</name>
  <files>internal/platform/auth/interceptor.go, internal/platform/auth/context.go, internal/platform/ratelimit/limiter.go, internal/platform/ratelimit/interceptor.go</files>
  <action>
    **Auth interceptor (internal/platform/auth/):**

    1. Create `context.go` with:
       - `Claims` struct: `UserID string`, `Username string` (extracted from JWT)
       - Context key type (unexported)
       - `WithClaims(ctx, *Claims) context.Context` — inject claims
       - `ClaimsFromContext(ctx) *Claims` — extract claims (nil if anonymous)
       - `UserIDFromContext(ctx) string` — convenience helper, returns "" if anonymous

    2. Create `interceptor.go` with:
       - `JWTValidator` struct holding `*jwt.Parser` and `secret []byte`
       - `NewJWTValidator(secret string) *JWTValidator`
       - `(v *JWTValidator) Validate(tokenString string) (*Claims, error)` — uses jwt.ParseWithClaims with HS256, issuer "redyx", expiration required
       - `publicMethods` map listing RPCs that allow anonymous access (from research: Register, Login, VerifyOTP, ResetPassword, GoogleOAuth, RefreshToken, GetProfile, GetUserPosts, GetUserComments, GetCommunity, ListCommunities, ListMembers, plus HealthService.Check)
       - `UnaryInterceptor(validator *JWTValidator) grpc.UnaryServerInterceptor` — extracts Bearer token from `authorization` metadata, validates, injects claims. Public methods proceed without auth. Protected methods without valid auth return `codes.Unauthenticated`
       - `extractToken(ctx) string` — gets token from gRPC metadata `authorization` header, strips "Bearer " prefix

    **Rate limit interceptor (internal/platform/ratelimit/):**

    3. Create `limiter.go` with:
       - Redis Lua token bucket script (as described in research Pattern 3 — atomic check+decrement)
       - `Limiter` struct with `*goredis.Client`
       - `New(rdb *goredis.Client) *Limiter`
       - `Result` struct: `Allowed bool`, `Remaining int`, `Limit int`, `RetryAfter time.Duration`
       - `(l *Limiter) Check(ctx, key string, limit int, windowSeconds int) (*Result, error)`
       - Tier configs as a package-level map:
         ```go
         var TierLimits = map[string]struct{ Limit, WindowSec int }{
             "anonymous":     {10, 60},
             "authenticated": {100, 60},
             "trusted":       {300, 60},
         }
         var ActionLimits = map[string]struct{ Limit, WindowSec int }{
             "post":      {5, 3600},
             "comment":   {30, 3600},
             "vote":      {60, 60},
             "community": {1, 86400},
         }
         ```

    4. Create `interceptor.go` with:
       - `UnaryInterceptor(limiter *Limiter, enabled bool) grpc.UnaryServerInterceptor`
       - If `!enabled`, pass through (no-op interceptor)
       - Extract user tier from auth claims in context: nil claims = "anonymous", otherwise "authenticated" (trusted tier deferred per research open question 5)
       - Build rate limit key: `rl:{tier}:{identifier}` where identifier is UserID or client IP from peer info
       - Call `limiter.Check` with tier limits
       - If not allowed, return `status.Error(codes.ResourceExhausted, "rate limit exceeded")` with trailing metadata `retry-after` set to seconds
       - If allowed, call handler and pass through
       - Note: Action-specific limits (post/comment/vote) will be called from individual service handlers, not the interceptor. The interceptor enforces global request rate only.
  </action>
