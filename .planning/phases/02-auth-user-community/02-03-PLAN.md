  <action>
    1. **server.go** — Implement all 5 RPCs:
       - `Server` struct: `*pgxpool.Pool`, `*zap.Logger`
       - `NewServer(db *pgxpool.Pool, logger *zap.Logger) *Server`
       - Embed `userv1.UnimplementedUserServiceServer`

       **GetProfile:** Query profiles table by username. If not found AND the request comes from an authenticated user whose username matches (from context claims), create the profile with defaults (create-on-first-access pattern — handles the gap between auth-service creating the user and profile existing). If not found at all, return ErrNotFound. If found but deleted_at is set, return a profile with username="[deleted]", empty bio/display_name/avatar, karma=0. Map DB row to User proto message.

       **UpdateProfile:** Get claims from context (required — protected endpoint). Query profile by user_id from claims. If not found, create it first (same create-on-first-access). Validate: bio max 500 chars (return ErrInvalidInput if exceeded), display_name max 50 chars. Update only non-empty fields (partial update pattern). Return updated User.

       **DeleteAccount:** Get claims from context. UPDATE profiles SET display_name='', bio='', avatar_url='', deleted_at=NOW() WHERE user_id=claims.UserID. This is a soft delete — username changes to "[deleted]" in the response but actual username stays in DB for uniqueness. NOTE: Full account deletion (auth DB cleanup) requires cross-service coordination — for Phase 2, this soft-deletes the profile only. Auth-service cleanup can be added later or the user can be marked in the auth users table too.
       For a more complete approach: also make a best-effort call to delete the auth record. Since services share Redis, publish a `user:deleted:{user_id}` message to a Redis channel that the auth service can listen for. OR just accept the limitation for Phase 2 and document it.

       **GetUserPosts:** Return empty `GetUserPostsResponse` with empty posts list and no pagination cursor. Log a debug message "GetUserPosts stub — post service not available until Phase 3".

       **GetUserComments:** Same as GetUserPosts — return empty list. Stub until Phase 4.

    2. **cmd/user/main.go** — Entry point following skeleton pattern:
       - Load config with `config.Load("user")`  (note: database URL default will be `user` but we need `user_profiles` — set DATABASE_URL env var to `postgres://redyx:dev@postgres:5432/user_profiles?sslmode=disable`)
       - Connect to PostgreSQL (user_profiles database)
       - Connect to Redis (for future use / rate limiting)
       - Run migrations: execute `migrations/user/*.up.sql` files
       - Create gRPC server: Recovery → Logging → RateLimit → AuthInterceptor → ErrorMapping
       - Register UserServiceServer
       - Run server on port from config (will be 50052 via GRPC_PORT env)
