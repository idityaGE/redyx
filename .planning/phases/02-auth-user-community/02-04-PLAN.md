  <name>Task 1: Create Redis community cache and implement CommunityServiceServer</name>
  <files>internal/community/cache.go, internal/community/server.go</files>
  <action>
    1. **cache.go** — Redis caching for community metadata:
       - `Cache` struct with `*goredis.Client` and `ttl time.Duration` (1 hour)
       - `NewCache(rdb *goredis.Client) *Cache`
       - `Get(ctx, name string) (*commv1.Community, error)` — gets `community:{name}` from Redis, JSON unmarshal. Returns nil,nil if miss.
       - `Set(ctx, name string, community *commv1.Community) error` — JSON marshal, SET with 1hr TTL
       - `Invalidate(ctx, name string) error` — DEL `community:{name}`
       - Use `encoding/json` for serialization (simple, proto messages are JSON-compatible via protojson if needed — or just cache the DB fields as a simpler struct and reconstruct the proto)

    2. **server.go** — Implement all 9 RPCs:
       - `Server` struct: `*pgxpool.Pool`, `*Cache`, `*zap.Logger`
       - `NewServer(db *pgxpool.Pool, cache *Cache, logger *zap.Logger) *Server`
       - Embed `commv1.UnimplementedCommunityServiceServer`

       **CreateCommunity:** Get claims from context (required). Validate name with regex `^[a-zA-Z0-9_]{3,21}$` (return ErrInvalidInput if fails). INSERT into communities table. INSERT into community_members (user_id=claims.UserID, role='owner'). UPDATE communities SET member_count=1. Return community. COMM-01 + COMM-05.

       **GetCommunity:** Check cache first. If miss, query DB by name. If not found, return ErrNotFound. Check visibility: if PRIVATE and user is not a member, return ErrForbidden. Populate is_member and is_moderator from community_members table if user is authenticated (claims != nil). Cache the result. Return. COMM-03 (visibility enforcement).

       **UpdateCommunity:** Get claims. Verify user is owner or moderator (query community_members for role). If not, return ErrForbidden. UPDATE only provided fields (description, rules as JSONB, banner_url, icon_url, visibility). Invalidate cache. Return updated community. COMM-02.

       **ListCommunities:** Query communities with cursor-based pagination using created_at as cursor. Support optional name query filter (ILIKE '%query%'). Order by member_count DESC as default sort. Use platform pagination helpers. Return communities list + pagination response.

       **JoinCommunity:** Get claims (required). Look up community by name. Check visibility: if PRIVATE, return ErrForbidden (invite-only — invite system deferred, for now private communities can't be joined). If RESTRICTED, allow join (restricted means non-members can view but not post — joining is still allowed). INSERT into community_members with role='member' (ON CONFLICT DO NOTHING for idempotency). UPDATE communities SET member_count = member_count + 1 (use a subquery counting actual members to prevent drift). COMM-04.

       **LeaveCommunity:** Get claims. Check user is member. If owner, return ErrForbidden with "owner cannot leave community" message. DELETE from community_members. UPDATE member_count. Invalidate cache. COMM-04.

       **ListMembers:** Query community_members JOIN with looking up usernames. Use cursor pagination. Return members with role and joined_at.

       **AssignModerator:** Get claims. Verify caller is owner (not just moderator — only owner can assign mods). Verify target user is a member. UPDATE community_members SET role='moderator' WHERE user_id=target AND community_id=community. Invalidate cache. COMM-06.

       **RevokeModerator:** Get claims. Verify caller is owner. Cannot revoke self. UPDATE community_members SET role='member'. Invalidate cache. COMM-06.

       **Note on usernames in ListMembers/Member responses:** The community DB stores user_id but not username. For Phase 2, store username alongside user_id in community_members (denormalized). This avoids a cross-service call to user-service for every member list query. Add a `username` column to community_members migration in plan 01's migrations.
       IMPORTANT: Check if the migration in plan 01 already includes username in community_members. If not, add it to community_members: `username TEXT NOT NULL DEFAULT ''`. The Join/AssignModerator operations get username from the auth claims context.
