  <name>Task 1: Create API client and auth store</name>
  <files>web/src/lib/api.ts, web/src/lib/auth.ts</files>
  <action>
    1. **api.ts** — Fetch wrapper following research Pattern 4:
       - `API_BASE = '/api/v1'` constant
       - Module-level `accessToken: string | null = null`
       - Module-level `refreshPromise: Promise<boolean> | null = null` (deduplication)
       - `setAccessToken(token: string | null)` — sets module-level token
       - `getAccessToken(): string | null` — returns current token
       - `refreshAccessToken(): Promise<boolean>` — POSTs to `/api/v1/auth/refresh` with current refreshToken from module-level storage. On success: sets new accessToken and refreshToken, returns true. On failure: returns false.
       - `api<T>(path: string, options?: RequestInit): Promise<T>` — main fetch wrapper:
         * Sets `Authorization: Bearer {token}` if token exists
         * Sets `Content-Type: application/json` if body present and no content-type set
         * On 401: deduplicates refresh call (singleton refreshPromise), retries original request on success, throws on failure
         * On non-ok: throws `ApiError` with status and message from JSON body
         * Returns parsed JSON as T
       - `ApiError` class extending Error with `status: number`
       - Module-level `refreshToken: string | null = null` for storing refresh token
       - `setRefreshToken(token: string | null)` — stores refresh token in memory (per research: not localStorage, not cookies — memory variable)

    2. **auth.ts** — Svelte 5 runes auth store following research Pattern 5:
       - NOTE: Svelte 5 runes (`$state`, `$derived`) can only be used in `.svelte` or `.svelte.ts` files. Name this file `auth.svelte.ts` instead of `auth.ts` for runes to work. Update all imports accordingly.
       - Actually, check Svelte 5 docs: module-level `$state` may need `.svelte.ts` extension. If so, rename to `web/src/lib/auth.svelte.ts`.
       - If `.svelte.ts` extension is required for runes in non-component files, use it. Otherwise, use a simple reactive pattern with getters/setters and a subscriber pattern.
       
       SAFER APPROACH: Use a plain `.ts` file with a simple pub/sub pattern that doesn't require runes:
       ```typescript
       type User = { userId: string; username: string } | null;
       type Subscriber = () => void;
       let user: User = null;
       let loading = true;
       const subscribers: Set<Subscriber> = new Set();
       
       function notify() { subscribers.forEach(fn => fn()); }
       export function subscribe(fn: Subscriber) { subscribers.add(fn); return () => subscribers.delete(fn); }
       export function getUser() { return user; }
       export function isAuthenticated() { return user !== null; }
       export function isLoading() { return loading; }
       ```
       
       ALTERNATIVE: Try `.svelte.ts` extension first. If it works in Astro's Svelte integration, prefer runes for consistency with component patterns.
       
       Exports:
       - `getUser()` — returns current user or null
       - `isAuthenticated()` — boolean
       - `isLoading()` — boolean (true during initial load)
       - `initialize()` — called once on app start. Attempts token refresh (cookie/stored token). If succeeds, fetches user profile and sets user state. If fails, sets user=null. Sets loading=false.
       - `login(email, password)` — calls `/auth/login`, sets tokens, fetches profile
       - `loginWithTokens(accessToken, refreshToken, userId)` — sets tokens after OAuth/OTP verification
       - `logout()` — calls `/auth/logout`, clears tokens and user state
       - `subscribe(callback)` — for non-runes reactivity if needed
