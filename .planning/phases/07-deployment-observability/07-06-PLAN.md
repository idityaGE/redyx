<task type="auto">
  <name>Task 1: Deploy full stack and run validation script</name>
  <files></files>
  <action>
Execute the full deployment sequence:

```bash
# 1. Create kind cluster and namespaces
make k8s-create

# 2. Build and load service images
make k8s-build
make k8s-load

# 3. Deploy data stores
make k8s-data

# 4. Deploy observability stack
make k8s-monitoring

# 5. Deploy application services
make k8s-app

# 6. Wait for all pods to be ready
kubectl wait --for=condition=ready pod --all -n redyx-app --timeout=300s
kubectl wait --for=condition=ready pod --all -n redyx-data --timeout=300s
kubectl wait --for=condition=ready pod --all -n redyx-monitoring --timeout=300s

# 7. Run validation script
./scripts/k8s-validate.sh
```

If any step fails, capture the error and attempt to diagnose:
- Pod not starting: `kubectl describe pod <pod> -n <namespace>`
- CrashLoopBackOff: `kubectl logs <pod> -n <namespace>`
- ImagePullBackOff: Verify `kind load docker-image` completed

Common issues and fixes:
- ScyllaDB slow start: Increase timeout, wait longer
- PostgreSQL init script: Check ConfigMap applied correctly
- Envoy routing: Check proto.pb is mounted, services reachable

<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
Complete Kubernetes deployment with:
- 12 microservices in redyx-app namespace
- 6 data stores in redyx-data namespace  
- Prometheus, Grafana, Loki, Jaeger in redyx-monitoring namespace
- Envoy API gateway with REST-to-gRPC transcoding
- HPA configured for all services
- ServiceMonitors for Prometheus scraping
- Grafana dashboards for all services
  </what-built>
  <how-to-verify>
**1. Check cluster status:**
```bash
make k8s-status
```
Verify all pods show "Running" status with 1/1 or appropriate ready count.

**2. Start port-forwards for observability UIs:**
```bash
make k8s-port
```

**3. Verify Prometheus targets (http://localhost:9090):**
- Navigate to Status → Targets
- All targets should show "UP" state
- Check for redyx-app services in the target list

**4. Verify Grafana dashboards (http://localhost:3000):**
- Login: admin/admin
- Navigate to Dashboards → Browse → Redyx folder
- Open "Redyx Global Overview" dashboard
- Verify panels show data (may be minimal if no traffic)
- Open one service dashboard (e.g., auth-service)
- Verify gRPC metrics panels are present

**5. Verify Loki logs (in Grafana):**
- Go to Explore → Select Loki datasource
- Run query: `{namespace="redyx-app"}`
- Verify logs appear from services

**6. Verify Jaeger traces (http://localhost:16686):**
- Select a service from dropdown (e.g., auth-service)
- Click "Find Traces"
- If no traces, make a test API call first:
  ```bash
  curl -X POST http://localhost:30080/api/v1/auth/register \
    -H "Content-Type: application/json" \
    -d '{"email":"test@example.com","username":"testuser","password":"Test123!"}'
  ```
- Return to Jaeger and verify trace appears

**7. Verify API gateway works:**
```bash
# Health check through Envoy
curl http://localhost:30080/api/v1/health

# Should return gRPC health response transcoded to JSON
```

**8. Verify HPA is configured:**
```bash
kubectl get hpa -n redyx-app
```
All services should have HPA with MINPODS/MAXPODS set.
  </how-to-verify>
