<task type="auto">
  <name>Task 2: Envoy comment routes, proto descriptor rebuild, cluster config</name>
  <files>deploy/envoy/envoy.yaml</files>
  <action>
    **1. Rebuild proto descriptor** to include CommentService:
    Run `make proto` — this regenerates `deploy/envoy/proto.pb` from all proto files including `comment.proto`. The descriptor must include `redyx.comment.v1.CommentService` for REST-to-gRPC transcoding.

    **2. Add comment routes to Envoy** — CRITICAL: These must come BEFORE the existing `/api/v1/posts` prefix route. The existing route order has `/api/v1/posts` at position #3 which would catch `/api/v1/posts/{post_id}/comments`. 

    Insert BEFORE the `- match: { prefix: "/api/v1/posts" }` line:
    ```yaml
    # Comment routes — BEFORE post catch-all (Pitfall 15)
    # Matches: GET/POST /api/v1/posts/{post_id}/comments
    - match:
        safe_regex:
          regex: "/api/v1/posts/[^/]+/comments.*"
      route:
        cluster: comment-service
        timeout: 30s
    # Matches: GET/PATCH/DELETE /api/v1/comments/{comment_id} and /api/v1/comments/{comment_id}/replies
    - match: { prefix: "/api/v1/comments" }
      route:
        cluster: comment-service
        timeout: 30s
    ```

    The regex route `/api/v1/posts/[^/]+/comments.*` catches:
    - `GET /api/v1/posts/{post_id}/comments` → ListComments
    - `POST /api/v1/posts/{post_id}/comments` → CreateComment
    
    The prefix route `/api/v1/comments` catches:
    - `GET /api/v1/comments/{comment_id}` → GetComment
    - `PATCH /api/v1/comments/{comment_id}` → UpdateComment
    - `DELETE /api/v1/comments/{comment_id}` → DeleteComment
    - `GET /api/v1/comments/{comment_id}/replies` → ListReplies

    **Final route order should be:**
    1. /api/v1/auth/ → auth-service
    2. /api/v1/users/ → user-service
    3. **NEW: /api/v1/posts/{post_id}/comments (regex) → comment-service**
    4. **NEW: /api/v1/comments → comment-service**
    5. /api/v1/posts → post-service
    6. /api/v1/feed → post-service
    7. /api/v1/saved → post-service
    8. /api/v1/votes → vote-service
    9. /api/v1/communities/{name}/posts (regex) → post-service
    10. /api/v1/communities → community-service
    11. /api/v1/ → skeleton-service

    **3. Add comment-service cluster:**
    ```yaml
    - name: comment-service
      type: STRICT_DNS
      lb_policy: ROUND_ROBIN
      typed_extension_protocol_options:
        envoy.extensions.upstreams.http.v3.HttpProtocolOptions:
          "@type": type.googleapis.com/envoy.extensions.upstreams.http.v3.HttpProtocolOptions
          explicit_http_config:
            http2_protocol_options: {}
      load_assignment:
        cluster_name: comment-service
        endpoints:
        - lb_endpoints:
          - endpoint:
              address:
                socket_address: { address: comment-service, port_value: 50057 }
    ```

    **4. Register CommentService in transcoder:**
    Add `- redyx.comment.v1.CommentService` to the services list in the grpc_json_transcoder config.
