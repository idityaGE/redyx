// JSON response from GET /api/v1/posts/{post_id}/comments
type ListCommentsResponse = {
  comments: Comment[];  // Flat list in display order (path-sorted with depth info)
  pagination: { nextCursor: string; hasMore: boolean; totalCount: number; };
};

type Comment = {
  commentId: string;
  postId: string;
  parentId: string;       // Empty for top-level
  authorId: string;
  authorUsername: string;  // "[deleted]" for deleted comments
  body: string;           // "[deleted]" for deleted comments, markdown otherwise
  voteScore: number;
  replyCount: number;
  path: string;           // "001", "001.002", etc.
  depth: number;          // 1 = top-level, 2 = reply, 3 = reply-to-reply
  isEdited: boolean;
  isDeleted: boolean;
  createdAt: string;      // ISO timestamp
  editedAt: string;
};

// JSON response from GET /api/v1/comments/{comment_id}/replies
type ListRepliesResponse = {
  replies: Comment[];
  pagination: { nextCursor: string; hasMore: boolean; };
};
```

    Props:
    - `comment: Comment` — the comment data
    - `postId: string` — for reply form API calls
    - `depth: number` — current nesting depth
    - `onReplySubmitted?: (comment: Comment, parentPath: string) => void` — bubble up new reply
    - `onDeleted?: (commentId: string) => void` — bubble up deletion

    - `replying: boolean` = $state(false) — inline reply form visible
    - `editing: boolean` = $state(false) — inline edit mode
    - `editBody: string` = $state(comment.body)
    - `confirmingDelete: boolean` = $state(false)
    - `localComment: Comment` = $state(comment) — for optimistic updates
    
    Layout structure:
    ```
    <div style="padding-left: {Math.min(depth - 1, 2) * 1.5}rem">
      {#if depth > 1}
        <div class="border-l border-terminal-border absolute left-0 top-0 bottom-0"></div>

        <span>·</span>
        <span>{relativeTime(localComment.createdAt)}</span>
        {#if localComment.isEdited}<span>(edited)</span>{/if}

          <span>· {localComment.replyCount} replies</span>
        {/if}
      </div>
      
      {#if !collapsed}
        <!-- Vote + body -->
        <div class="flex gap-2">
          {#if !localComment.isDeleted}

              targetType="TARGET_TYPE_COMMENT"
              initialScore={localComment.voteScore}
              initialVote={0}
            />
          {:else}
            <span class="text-terminal-dim w-8 text-center">{localComment.voteScore}</span>
          {/if}
          
          <div class="flex-1">
            {#if editing}
              <!-- Inline edit textarea -->
              <textarea bind:value={editBody} class="w-full font-mono bg-terminal-bg border border-terminal-border p-2 text-terminal-text" />
              <div class="flex gap-2 mt-1">
                <button onclick={submitEdit}>[save]</button>
                <button onclick={() => editing = false}>[cancel]</button>
              </div>
            {:else if localComment.isDeleted}
              <p class="text-terminal-dim italic">[deleted]</p>
            {:else}
              <PostBody body={localComment.body} />

            <!-- Action bar (only for non-deleted, authenticated) -->
            {#if !localComment.isDeleted && !editing}

                      <button onclick={() => confirmingDelete = true}>[delete]</button>

    **handleDelete:**
    1. Optimistic: set localComment body to "[deleted]", author to "[deleted]", isDeleted to true
    2. Call `DELETE /api/v1/comments/${comment.commentId}`
    3. On error: revert localComment to original
    4. Call onDeleted callback
    
    **handleReplySubmitted(newComment):**
    - Call onReplySubmitted to bubble up to CommentSection for list insertion
    - Set replying=false
    
    **submitEdit:**
    1. Call `PATCH /api/v1/comments/${comment.commentId}` with `{ body: editBody }`
    2. On success: update localComment.body and isEdited, set editing=false
    3. On error: show error

    **2. Create CommentSection** at `web/src/components/comment/CommentSection.svelte`:
    
    Props:
    - `postId: string`
    
    State:
    - `comments: Comment[]` = $state([]) — flat list in display order
    - `sort: string` = $state(localStorage commentSort or "COMMENT_SORT_ORDER_BEST")
    - `loading: boolean` = $state(true)
    - `error: string` = $state("")
    - `hasMore: boolean` = $state(false)
    - `nextCursor: string` = $state("")
    - `totalCount: number` = $state(0)
    - `loadingMore: boolean` = $state(false)
    
    On mount:
    ```typescript
    import { whenReady } from '../../lib/auth';
    import { onMount } from 'svelte';
    
    onMount(() => {
      whenReady().then(() => fetchComments());
    });
    ```
    
    **fetchComments(cursor?: string):**
    1. Set loading (or loadingMore if cursor) = true
    2. GET `/api/v1/posts/${postId}/comments?sort=${sort}&pagination.limit=20${cursor ? '&pagination.cursor=' + cursor : ''}`
    3. On success: 
       - If no cursor: set comments = response.comments
       - If cursor: append response.comments to existing
       - Set hasMore, nextCursor, totalCount from pagination
    4. On error: set error
    5. Set loading/loadingMore = false
    
    **Sort change handler:**
    - Save to localStorage as `commentSort`
    - Reset comments, cursor
    - Call fetchComments() (no cursor = fresh fetch)
    - Use `untrack()` pattern if using $effect for sort reactivity
    
    **handleNewTopLevelComment(comment):**
    - Prepend to comments array (appears at top regardless of sort — optimistic)
    - Increment totalCount
    
    **handleReplySubmitted(newComment, parentPath):**
    - Find insertion index: after the parent comment's subtree in the flat list
    - Insert newComment at correct position
    - Update parent's replyCount in the local comments array
    
    **handleLoadMoreReplies(commentId):**
    - GET `/api/v1/comments/${commentId}/replies?pagination.limit=20`
    - Insert returned replies at correct position in flat comments array (after parent's existing children)
    
    Layout:
    ```svelte
    <div class="mt-6 border-t border-terminal-border pt-4">
      <div class="flex items-center gap-4 mb-4">
        <h3 class="text-terminal-text font-mono">{totalCount} comments</h3>
        <CommentSortBar sort={sort} onSortChange={handleSortChange} />
      </div>
      
      <CommentForm 
        postId={postId}
        isTopLevel={true}
        onSubmit={handleNewTopLevelComment}
      />
      
      {#if loading}
        <div class="text-terminal-dim font-mono py-4">loading comments...</div>
      {:else if error}
        <div class="text-terminal-error font-mono py-4">> error: {error}</div>
      {:else if comments.length === 0}
        <div class="text-terminal-dim font-mono py-8 text-center">no comments yet. be the first.</div>
      {:else}
        {#each comments as comment (comment.commentId)}
          <CommentCard 
            comment={comment}
            postId={postId}
            depth={comment.depth}
            onReplySubmitted={handleReplySubmitted}
            onDeleted={handleCommentDeleted}
          />
          
          <!-- [load N more replies] for comments with hidden children -->
          {#if comment.depth <= 3 && comment.replyCount > 0 && !hasLoadedReplies(comment)}
            <!-- Only show if this comment has replies but they weren't included in the response -->
          {/if}
        {/each}
        
        {#if hasMore}
          <button onclick={() => fetchComments(nextCursor)} 
                  class="text-terminal-accent font-mono text-sm py-2">
            {loadingMore ? 'loading...' : '[load more comments]'}
          </button>
        {/if}
      {/if}
    </div>
    ```

    **Deep thread [load more replies] logic:**
    - For each comment in the flat list where depth >= 3 and replyCount > 0:
    - Check if child comments are already in the list (next items with matching path prefix)
    - If not loaded: show `[load {replyCount} more replies]` button
    - On click: call handleLoadMoreReplies(commentId)
    - Insert loaded replies at the correct position in the flat array
    
    **Handling the flat list display order:**
    The server returns comments in tree-display order (top-level sorted by chosen sort, their children in path order up to depth 3). The frontend renders them flat with depth-based indentation via CommentCard's depth prop. No tree-building needed — just iterate the flat array.
