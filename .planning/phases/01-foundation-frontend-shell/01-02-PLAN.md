<task type="auto">
  <name>Task 1: Initialize Astro project with Svelte, Node adapter, and TailwindCSS v4</name>
  <files>web/package.json, web/astro.config.mjs, web/tsconfig.json, web/tailwind.css, web/src/styles/terminal.css, web/public/fonts/</files>
  <action>
1. Create the `web/` directory and initialize the Astro project:
   ```bash
   cd web
   npm create astro@latest -- . --template minimal --no-install --typescript strict
   ```
   If the above doesn't work non-interactively, create the project files manually.

2. Install dependencies:
   ```bash
   npm install astro @astrojs/svelte @astrojs/node svelte
   npm install -D tailwindcss @tailwindcss/vite
   ```

3. Configure `web/astro.config.mjs`:
   ```javascript
   import { defineConfig } from 'astro/config';
   import svelte from '@astrojs/svelte';
   import node from '@astrojs/node';
   import tailwindcss from '@tailwindcss/vite';

   export default defineConfig({
     output: 'server',
     adapter: node({ mode: 'standalone' }),
     integrations: [svelte()],
     vite: {
       plugins: [tailwindcss()],
     },
   });
   ```

   CRITICAL: `output: 'server'` is required for SSR. Without it, Astro defaults to static site generation and dynamic pages break.

4. Create `web/tailwind.css` with the terminal theme (TailwindCSS v4 CSS-first config):
   ```css
   @import "tailwindcss";

   @theme {
     --font-mono: 'JetBrains Mono', 'Fira Code', 'Cascadia Code', ui-monospace, monospace;

     --color-accent-50: #fff7ed;
     --color-accent-100: #ffedd5;
     --color-accent-200: #fed7aa;
     --color-accent-300: #fdba74;
     --color-accent-400: #fb923c;
     --color-accent-500: #f97316;
     --color-accent-600: #ea580c;
     --color-accent-700: #c2410c;
     --color-accent-800: #9a3412;
     --color-accent-900: #7c2d12;

     --color-terminal-bg: #0a0a0a;
     --color-terminal-fg: #d4d4d4;
     --color-terminal-border: #262626;
     --color-terminal-dim: #737373;
     --color-terminal-surface: #171717;
   }
   ```

5. Create `web/src/styles/terminal.css` with ASCII-style utility classes:
   ```css
   /* Box-drawing border utilities and terminal-style patterns */
   @utility border-terminal {
     border: 1px solid var(--color-terminal-border);
   }

   @utility box-terminal {
     border: 1px solid var(--color-terminal-border);
     background-color: var(--color-terminal-surface);
     padding: 0.5rem;
     font-family: var(--font-mono);
   }

   @utility text-terminal-dim {
     color: var(--color-terminal-dim);
   }

   @utility bg-terminal {
     background-color: var(--color-terminal-bg);
   }

   @utility bg-terminal-surface {
     background-color: var(--color-terminal-surface);
   }
   ```
   Import this file in tailwind.css after the @import "tailwindcss" line.

6. Download and self-host JetBrains Mono font (WOFF2 format) into `web/public/fonts/`. Alternatively, use Bunny Fonts CDN link in the base layout (privacy-friendly alternative to Google Fonts). Per research recommendation, use Bunny Fonts CDN for simplicity: `https://fonts.bunny.net/css?family=jetbrains-mono:400,500,600,700`

7. Ensure tsconfig.json is configured for strict TypeScript.

IMPORTANT: Do NOT use `@astrojs/tailwind` integration — it's deprecated. Use `@tailwindcss/vite` plugin directly per research. Do NOT create `tailwind.config.js` — TailwindCSS v4 uses CSS-first config.
  </action>

<task type="auto">
  <name>Task 3: Add Svelte interactive islands (MobileNav, ThemeToggle) and responsive behavior</name>
  <files>web/src/components/MobileNav.svelte, web/src/components/ThemeToggle.svelte, web/src/layouts/BaseLayout.astro</files>
  <action>
Create the Svelte 5 interactive components and wire responsive behavior into the layout.

**Locked decisions:**
- On mobile: bottom tab bar (Home, Communities, Search, Notifications, Profile) instead of hamburger menu
- Sidebar collapses on tablet, bottom tabs on mobile phone widths
- Both dark and light modes are first-class
- Dark mode should be the default experience

**MobileNav.svelte** — Bottom tab bar (Svelte 5 with runes):
- Use Svelte 5 runes API ($state, $derived) — NOT Svelte 4 stores
- 5 tabs: Home (⌂), Communities (◈), Search (⌕), Notifications (♦), Profile (◉)
- Use simple Unicode characters for icons to match terminal aesthetic — NOT SVG icons
- Active tab highlighted with accent-500 color
- Track active tab with `$state`
- Fixed to bottom of screen, full width
- bg-terminal-surface, border-terminal on top
- Each tab: flex-col items-center, text-xs font-mono
- Compact: h-14
- Only visible on screens < lg (1024px) — controlled by parent CSS class

**ThemeToggle.svelte** — Dark/light mode switch (Svelte 5 with runes):
- Use Svelte 5 runes API
- Toggle button showing `[dark]` / `[light]` text (terminal style, not icons)
- On click: toggles `dark` class on `<html>` element
- Persists preference to localStorage
- On mount: reads localStorage, applies saved preference (default: dark)
- Positioned fixed, bottom-right corner on desktop (bottom-4 right-4), hidden on mobile (where MobileNav is shown)
- Small, unobtrusive button with border-terminal styling

**Update BaseLayout.astro:**
- Add MobileNav with `client:visible` directive (hydrates when scrolled into view — appropriate for bottom nav)
- Add ThemeToggle with `client:load` directive (hydrates immediately — needs to prevent flash of wrong theme)
- MobileNav wrapper: `<div class="lg:hidden fixed bottom-0 left-0 right-0 z-50">`
- ThemeToggle wrapper: `<div class="hidden lg:block fixed bottom-4 right-4 z-50">`
- Add inline script in `<head>` to prevent flash of light mode:
  ```html
  <script is:inline>
    if (localStorage.getItem('theme') !== 'light') {
      document.documentElement.classList.add('dark');
    } else {
      document.documentElement.classList.remove('dark');
    }
  </script>
  ```

**Responsive breakpoints (TailwindCSS defaults):**
- Mobile: < 768px (sm) → Bottom tab bar, no sidebar, no footer
- Tablet: 768px - 1023px (md) → No sidebar, no bottom tabs, footer visible
- Desktop: ≥ 1024px (lg) → Sidebar visible, footer visible, no bottom tabs

**Light mode styles:**
- TailwindCSS v4 supports `dark:` variant. Structure classes so:
  - Dark (default): bg-terminal (#0a0a0a), text-terminal-fg (#d4d4d4), borders #262626
  - Light: bg-white, text-neutral-900, borders neutral-200
- Apply dark-mode-first approach: base styles are dark, `light` overrides via removing `dark` class

Verify the layout is responsive by checking the build succeeds and the CSS classes are correct for all breakpoints.
