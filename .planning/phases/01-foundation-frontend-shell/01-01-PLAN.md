   ```yaml
   version: v2
   clean: true
   managed:
     enabled: true
     override:
       - file_option: go_package_prefix

   plugins:
     - remote: buf.build/protocolbuffers/go
       out: gen
       opt: paths=source_relative
     - remote: buf.build/grpc/go
       out: gen
       opt: paths=source_relative
   inputs:
     - directory: proto
   ```

Create all 14 proto files. Each must:
- Use `syntax = "proto3";`
- Have correct package name (e.g., `package redyx.health.v1;`)
- Import `google/api/annotations.proto` for HTTP annotations
- Include `google.api.http` option on every RPC with REST path using `/api/v1/` prefix
- Use snake_case field names (proto convention; Envoy transcodes to camelCase)
- Use `google.protobuf.Timestamp` for time fields

**Discretion decision (from CONTEXT.md):** JSON field naming = camelCase. Proto fields stay snake_case, Envoy `preserve_proto_field_names: false` (default) outputs camelCase. This is the proto3 canonical mapping.

**API path convention (locked from CONTEXT.md):** Flat resource paths: `/api/v1/posts`, `/api/v1/communities`, `/api/v1/users`, etc. Query params for filtering.

**common.proto** ã Shared types:
- `PaginationRequest` (string cursor, int32 limit)
- `PaginationResponse` (string next_cursor, bool has_more, int32 total_count)

**health.proto** ã Full implementation (Phase 1 skeleton):
- `HealthService.Check` ã `GET /api/v1/health`
- `HealthCheckRequest` (string service)
- `HealthCheckResponse` (ServingStatus enum: UNSPECIFIED/SERVING/NOT_SERVING, string version, google.protobuf.Timestamp uptime)

**For the remaining 12 service protos:** Define service RPCs with HTTP annotations and complete message types based on REQUIREMENTS.md. These are the API contracts for all future phases. Key RPCs per service:

- **auth.proto**: Register, Login, RefreshToken, Logout, VerifyOTP, ResetPassword, GoogleOAuth
- **user.proto**: GetProfile, UpdateProfile, DeleteAccount, GetUserPosts, GetUserComments
- **community.proto**: CreateCommunity, GetCommunity, UpdateCommunity, ListCommunities, JoinCommunity, LeaveCommunity, ListMembers, AssignModerator, RevokeModerator
- **post.proto**: CreatePost, GetPost, UpdatePost, DeletePost, ListPosts (community feed), ListHomeFeed, SavePost, ListSavedPosts
- **comment.proto**: CreateComment, GetComment, UpdateComment, DeleteComment, ListComments (by post), ListReplies
- **vote.proto**: Vote (upvote/downvote/remove), GetVoteState
- **search.proto**: SearchPosts, AutocompleteCommunities
- **media.proto**: InitUpload, CompleteUpload, GetMedia
- **notification.proto**: ListNotifications, MarkRead, MarkAllRead, GetPreferences, UpdatePreferences
- **moderation.proto**: RemoveContent, BanUser, UnbanUser, PinPost, UnpinPost, GetModLog, ListReportQueue
- **ratelimit.proto**: CheckRateLimit
- **spam.proto**: CheckContent, ReportSpam

Each service proto should have complete request/response messages with all fields from REQUIREMENTS.md. Use sensible field numbers (1-15 for frequently accessed fields for smaller wire size).

<task type="auto">
  <name>Task 3: Create Makefile and run proto generation</name>
  <files>Makefile, gen/, deploy/envoy/proto.pb</files>
  <action>
1. Create `Makefile` with these targets:

```makefile
.PHONY: proto proto-lint proto-breaking proto-descriptor build test clean help

proto: proto-lint  ## Generate Go code + Envoy descriptor from protos
	buf generate
	buf build -o deploy/envoy/proto.pb

proto-lint:  ## Lint proto files
	buf lint

proto-breaking:  ## Check for breaking changes vs git main
	buf breaking --against '.git#branch=main'

proto-descriptor:  ## Build Envoy descriptor set only
	buf build -o deploy/envoy/proto.pb

build:  ## Build all services
	go build ./cmd/...

test:  ## Run all tests
	go test ./...

clean:  ## Remove generated files
	rm -rf gen/
	rm -f deploy/envoy/proto.pb

help:  ## Show this help
	@grep -E '^[a-zA-Z_-]+:.*?## .*$$' $(MAKEFILE_LIST) | sort | awk 'BEGIN {FS = ":.*?## "}; {printf "\033[36m%-20s\033[0m %s\n", $$1, $$2}'

.DEFAULT_GOAL := help
```

2. Create the `deploy/envoy/` directory (for proto.pb output).

3. Run `make proto` to:
   - Lint all proto files
   - Generate Go code into gen/
   - Build Envoy descriptor set to deploy/envoy/proto.pb

4. Run `go mod tidy` after generation to clean up dependencies.

5. Verify:
   - gen/ directory contains .pb.go and _grpc.pb.go files for every service
   - deploy/envoy/proto.pb exists and is non-empty
   - Go code compiles: `go build ./gen/...`
