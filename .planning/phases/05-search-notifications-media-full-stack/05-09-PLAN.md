<task type="auto">
  <name>Task 1: API integration tests for search, notifications, and media</name>
  <files></files>
  <action>
Run `docker compose up --build -d` to bring up all services, then test each feature via curl:

**Search Service Tests:**
1. Wait for Meilisearch to be healthy: `curl -s http://localhost:7700/health`
2. Create a test post via the post-service API (reuse auth flow: register → login → create post)
3. Wait 2-3 seconds for Kafka → search indexer to process
4. Search for the post: `curl -s http://localhost:8080/api/v1/search/posts?query={title_keyword}` — expect the post in results within 300ms
5. Search with community scope: `curl -s http://localhost:8080/api/v1/search/posts?query={keyword}&communityName={name}` — expect scoped results
6. Autocomplete communities: `curl -s http://localhost:8080/api/v1/search/communities?query={prefix}&limit=5` — expect community suggestions

**Notification Service Tests:**
7. Create a comment on the test post (as a different user or same user)
8. Wait 1-2 seconds for Kafka → notification consumer
9. List notifications: `curl -s -H "Authorization: Bearer {token}" http://localhost:8080/api/v1/notifications` — expect notification for post reply
10. Get preferences: `curl -s -H "Authorization: Bearer {token}" http://localhost:8080/api/v1/notifications/preferences` — expect default preferences
11. Mark notification as read: `curl -s -X POST -H "Authorization: Bearer {token}" http://localhost:8080/api/v1/notifications/{id}/read`
12. Mark all read: `curl -s -X POST -H "Authorization: Bearer {token}" http://localhost:8080/api/v1/notifications/read-all`

**Media Service Tests:**
13. Init upload: `curl -s -X POST -H "Authorization: Bearer {token}" -H "Content-Type: application/json" -d '{"filename":"test.jpg","contentType":"image/jpeg","sizeBytes":1024,"mediaType":"MEDIA_TYPE_IMAGE"}' http://localhost:8080/api/v1/media/upload` — expect mediaId and uploadUrl
14. Upload a test file to the presigned URL: `curl -s -X PUT -H "Content-Type: image/jpeg" --data-binary @test_image.jpg {uploadUrl}` (create a small test image first)
15. Complete upload: `curl -s -X POST -H "Authorization: Bearer {token}" http://localhost:8080/api/v1/media/{mediaId}/complete` — expect url and status READY
16. Get media: `curl -s http://localhost:8080/api/v1/media/{mediaId}` — expect metadata with url

**WebSocket Test:**
17. Test WebSocket connection: Use `websocat` or a simple script to connect to `ws://localhost:8080/api/v1/ws/notifications?token={jwt}` — verify connection established

Fix any issues found during testing (misrouted requests, missing env vars, serialization errors, etc.).
