<interfaces>
<!-- Proto contract for media service -->
From proto/redyx/media/v1/media.proto:
- InitUpload(filename, content_type, size_bytes, media_type) → (media_id, upload_url, expires_at)
- CompleteUpload(media_id) → (url, thumbnail_url, status)
- GetMedia(media_id) → (media_id, url, thumbnail_url, media_type, status, content_type, size_bytes, created_at)
- MediaType: UNSPECIFIED, IMAGE, VIDEO, GIF
- MediaStatus: UNSPECIFIED, PENDING, PROCESSING, READY, FAILED

<!-- Presigned URL flow from research -->
1. Client → InitUpload (validates type/size, creates PENDING record, returns presigned PUT URL)
2. Client → PUT upload_url (direct to MinIO with file bytes)
3. Client → CompleteUpload (verifies object in S3, triggers thumbnail, updates to READY)
- UsePathStyle: true for MinIO (CRITICAL — virtual-hosted style won't work)
- Presigned URL expiry: 1 hour for video uploads (Pitfall 6)
</interfaces>

  <files>migrations/media/001_media.up.sql, internal/media/store.go, internal/media/s3.go, internal/media/thumbnail.go</files>
  <action>
1. Create `migrations/media/001_media.up.sql`:
   ```sql
   CREATE TABLE IF NOT EXISTS media_items (
     id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
     user_id UUID NOT NULL,
     filename VARCHAR(255) NOT NULL,
     content_type VARCHAR(100) NOT NULL,
     size_bytes BIGINT NOT NULL,
     media_type VARCHAR(20) NOT NULL,  -- 'image', 'video', 'gif'
     status VARCHAR(20) NOT NULL DEFAULT 'pending',  -- pending, processing, ready, failed
     s3_key VARCHAR(500) NOT NULL,
     url TEXT,
     thumbnail_url TEXT,
     error_message TEXT,
     created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
     updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
   );
   CREATE INDEX IF NOT EXISTS idx_media_items_user ON media_items(user_id, created_at DESC);
   CREATE INDEX IF NOT EXISTS idx_media_items_status ON media_items(status);
   ```

2. Create `internal/media/store.go`:
   - `Store` struct with `*pgxpool.Pool` and `*zap.Logger`
   - `NewStore(pool *pgxpool.Pool, logger *zap.Logger) *Store`
   - `Create(ctx, item MediaItem) (string, error)` — INSERT returning ID. MediaItem has: userID, filename, contentType, sizeBytes, mediaType, status, s3Key
   - `Get(ctx, mediaID string) (*MediaItem, error)` — SELECT by ID
   - `UpdateStatus(ctx, mediaID, status, url, thumbnailURL, errorMsg string) error` — UPDATE status + URLs
   - Internal MediaItem struct mapping to DB columns

3. Create `internal/media/s3.go`:
   - `S3Client` struct wrapping `*s3.Client` and bucket name
   - `NewS3Client(endpoint, accessKey, secretKey, region, bucket string) (*S3Client, error)`:
     - Creates s3.Client with:
       - `BaseEndpoint: aws.String(endpoint)` (e.g. "http://minio:9000")
       - `UsePathStyle: true` (CRITICAL for MinIO per Pitfall 2)
       - `Region: region` (default "us-east-1")
       - `Credentials: credentials.NewStaticCredentialsProvider(accessKey, secretKey, "")`
   - `GeneratePresignedPUT(ctx, key, contentType string, sizeBytes int64) (string, time.Time, error)`:
     - Uses `s3.NewPresignClient` to create presigned PutObject URL
     - Expiry: 1 hour (long enough for large video uploads per Pitfall 6)
     - Returns URL + expiry time
   - `ObjectExists(ctx, key string) (bool, error)` — HeadObject to verify upload completed
   - `GetObjectURL(key string) string` — constructs the public URL for the object (MinIO endpoint + bucket + key)
   - `GetObject(ctx, key string) (io.ReadCloser, error)` — GetObject to download for thumbnail generation

4. Create `internal/media/thumbnail.go`:
   - `GenerateThumbnail(ctx context.Context, s3Client *S3Client, sourceKey, thumbKey string) error`:
     - Download source image from S3 via s3Client.GetObject
     - Decode with `imaging.Decode`
     - Resize to max 320px wide maintaining aspect ratio with `imaging.Resize(img, 320, 0, imaging.Lanczos)`
     - Encode as JPEG and upload thumbnail back to S3
   - Only processes images (JPEG, PNG, GIF, WebP) — skip video thumbnails for v1 (per research Open Question 6: video thumbnails deferred, return empty thumbnail_url for videos)
   - Handle errors gracefully: if thumbnail fails, set status to READY anyway with empty thumbnail_url (don't block the upload)

<task type="auto">
  <name>Task 2: Create media-service gRPC server and entry point</name>
  <files>internal/media/server.go, cmd/media/main.go</files>
  <action>
1. Create `internal/media/server.go`:
   - `Server` struct with `store *Store`, `s3 *S3Client`, `logger *zap.Logger`
   - `NewServer(store *Store, s3 *S3Client, logger *zap.Logger) *Server`
   - Implement `InitUpload` RPC:
     - Extract user_id from context (auth required)
     - Validate content_type against allowed types:
       - Images: image/jpeg, image/png, image/gif, image/webp (max 20MB)
       - Videos: video/mp4, video/webm (max 100MB)
     - Return InvalidArgument for disallowed types or size exceeding limits (per MDIA-02)
     - Generate S3 key: `media/{user_id}/{uuid}/{filename}` (unique path prevents collisions)
     - Create PENDING media record in store
     - Generate presigned PUT URL via s3Client
     - Return InitUploadResponse with media_id, upload_url, expires_at
   - Implement `CompleteUpload` RPC:
     - Extract user_id from context
     - Get media record from store, verify ownership (user_id matches)
     - Verify object exists in S3 via s3Client.ObjectExists (confirms upload completed)
     - If object not found, return NotFound error
     - Update status to PROCESSING
     - Generate thumbnail (if image type): call GenerateThumbnail in a goroutine with thumb key `media/{user_id}/{uuid}/thumb_{filename}`
     - For synchronous v1: generate thumbnail inline (blocking) to simplify — it's fast for images.
     - Update status to READY with url and thumbnail_url
     - Return CompleteUploadResponse with public URLs and status
   - Implement `GetMedia` RPC:
     - Get media record by ID from store
     - Map to proto GetMediaResponse
     - This is a public endpoint (no auth required) — anyone can fetch media metadata for display

2. Create `cmd/media/main.go`:
   - Follow pattern from `cmd/comment/main.go`
   - Load config for "media" service
   - Connect to PostgreSQL (media database)
   - Run migrations from `migrations/media/`
   - Connect to Redis DB 9 (media-service's reserved DB)
   - Create S3Client from config (MinIOEndpoint, MinIOAccessKey, MinIOSecretKey, MinIOBucket)
   - Create Store, Server
   - Create gRPC server with middleware chain on port 50060
   - Register MediaServiceServer
   - Block on srv.Run()
