<interfaces>
<!-- API endpoints (from proto) -->
POST /api/v1/media/upload { filename, contentType, sizeBytes, mediaType }
→ { mediaId, uploadUrl, expiresAt }

POST /api/v1/media/{media_id}/complete → { url, thumbnailUrl, status }

GET /api/v1/media/{media_id}
→ { mediaId, url, thumbnailUrl, mediaType, status, contentType, sizeBytes, createdAt }

<!-- Upload flow -->
1. Call InitUpload → get presigned URL
2. PUT file bytes to presigned URL (XMLHttpRequest for progress events, NOT fetch — fetch lacks upload progress)
3. Call CompleteUpload → get final URL + thumbnail

<!-- User locked decisions -->
- Multiple files: up to 4-5 images OR 1 video (no mixing)
- Drag-and-drop zone: dashed border, terminal text "[ drop files here or click to browse ]"
- Per-file progress bar: ASCII `[=========>   ] 78%` with filename and size
- Thumbnail previews with [x] remove
- Submit during processing allowed (PENDING/PROCESSING status)
- Inline per-file errors: `> error: file too large (23MB / 20MB max)` in red
- Stacked images with lightbox in post detail view
- One video, no mixing: if video selected, takes full slot
</interfaces>

1. Create `web/src/components/media/MediaUpload.svelte`:
   - Props: `onMediaChange: (mediaIds: string[]) => void` — callback to pass uploaded media IDs to parent (PostSubmitForm)
   - State:
     - `let files = $state([])` — array of { id: string, file: File, progress: number, status: 'uploading'|'complete'|'error', mediaId: string|null, url: string|null, thumbnailUrl: string|null, error: string|null }
     - `let dragOver = $state(false)`
   - Drop zone:
     - Dashed border area (`border-2 border-dashed border-terminal-border`) with terminal text: `[ drop files here or click to browse ]` in `text-terminal-dim`
     - On dragover/dragleave: toggle `dragOver` state for visual feedback (border color change to accent)
     - On drop: process dropped files
     - Hidden `<input type="file" multiple accept="image/jpeg,image/png,image/gif,image/webp,video/mp4,video/webm">` triggered by clicking the zone
   - File processing:
     - Client-side validation BEFORE upload:
       - Images: check type is image/jpeg|png|gif|webp AND size <= 20MB (20 * 1024 * 1024 bytes)
       - Videos: check type is video/mp4|webm AND size <= 100MB
       - If invalid, set file.error = `"> error: file too large (23MB / 20MB max)"` or `"> error: unsupported format"` and file.status = 'error'
     - Enforce mixing rule: if any file is video, only 1 file allowed. If images, max 5. If user tries to add video when images exist (or vice versa), show error.
   - Upload flow per valid file:
     a. Call `api('/api/v1/media/upload', { method: 'POST', body: { filename, contentType, sizeBytes, mediaType } })` to get presigned URL
     b. Use XMLHttpRequest (NOT fetch) for PUT to presigned URL — XHR supports upload.onprogress:
        ```typescript
        const xhr = new XMLHttpRequest();
        xhr.open('PUT', uploadUrl);
        xhr.setRequestHeader('Content-Type', file.type);
        xhr.upload.onprogress = (e) => {
          if (e.lengthComputable) files[i].progress = Math.round((e.loaded / e.total) * 100);
        };
        xhr.onload = () => { /* call CompleteUpload */ };
        xhr.onerror = () => { files[i].error = '> error: upload failed'; files[i].status = 'error'; };
        xhr.send(file);
        ```
     c. On XHR success, call `api('/api/v1/media/{mediaId}/complete', { method: 'POST' })` to trigger thumbnail + get final URLs
     d. Update file entry with status='complete', url, thumbnailUrl
     e. Call onMediaChange with all completed mediaIds
   - Progress bar display: For each uploading file, show:
     ```
     filename.jpg (2.3 MB)
     [=========>        ] 54%
     ```
     The progress bar is ASCII: `[` + `=` repeated for filled portion + `>` at tip + spaces for remaining + `]` + percent. Terminal monospace font.
   - Error display: Red text `> error: {message}` below the file entry

2. Create `web/src/components/media/MediaPreview.svelte`:

     - For images: show the thumbnailUrl from CompleteUpload response
   - Click `[x]`: calls onRemove, which removes the file from the list and calls onMediaChange with updated mediaIds
