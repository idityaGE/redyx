       ```
       mc alias set local http://minio:9000 minioadmin minioadmin &&
       mc mb local/redyx-media --ignore-existing &&
       mc anonymous set download local/redyx-media
       ```
     - (CORS for PUT uploads from browser per Pitfall 2 in research — MinIO needs CORS config. Use mc to set CORS via `mc cors set` or environment. For dev, MinIO allows all origins by default when accessed directly — the presigned URL includes auth, so CORS is less of an issue for PUT. But set it explicitly to be safe.)

2. Update `deploy/envoy/envoy.yaml` routes — add BEFORE the skeleton catch-all route, and note ordering:
   - WebSocket route MUST come BEFORE the gRPC transcoder processes it (per Pitfall 1 from research). Add as first route:
     ```yaml
     # WebSocket for notifications — BEFORE all other routes
     # Must bypass gRPC transcoder (WebSocket is raw HTTP/1.1 upgrade)
     - match: { prefix: "/api/v1/ws/notifications" }
       route:
         cluster: notification-ws
         timeout: 0s  # No timeout for WebSocket (long-lived connection)
         upgrade_configs:
           - upgrade_type: websocket
     ```
   - Search routes:
     ```yaml
     - match: { prefix: "/api/v1/search" }
       route:
         cluster: search-service
         timeout: 30s
     ```
   - Notification routes (BEFORE the catch-all):
     ```yaml
     - match: { prefix: "/api/v1/notifications" }
       route:
         cluster: notification-service
         timeout: 30s
     ```
   - Media routes:
     ```yaml
     - match: { prefix: "/api/v1/media" }
       route:
         cluster: media-service
         timeout: 30s
     ```

3. Add new clusters:
   - `search-service`: HTTP/2 (gRPC), port 50058
   - `notification-service`: HTTP/2 (gRPC), port 50059
   - `media-service`: HTTP/2 (gRPC), port 50060
   - `notification-ws`: HTTP/1.1 (NOT HTTP/2 — WebSocket requires HTTP/1.1 upgrade per Pitfall 1), port 8081
     ```yaml
     - name: notification-ws
       type: STRICT_DNS
       lb_policy: ROUND_ROBIN
       # CRITICAL: WebSocket requires HTTP/1.1, NOT HTTP/2
       load_assignment:
         cluster_name: notification-ws
         endpoints:
         - lb_endpoints:
           - endpoint:
               address:
                 socket_address: { address: notification-service, port_value: 8081 }
     ```

4. Add new services to gRPC transcoder services list:
   - `redyx.search.v1.SearchService`
   - `redyx.notification.v1.NotificationService`
   - `redyx.media.v1.MediaService`

5. Route ordering check — final order should be:
