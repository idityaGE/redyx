<task type="auto">
  <name>Task 1: Create WebSocket client, NotificationBell, and NotificationDropdown</name>
  <files>web/src/lib/websocket.ts, web/src/components/notification/NotificationBell.svelte, web/src/components/notification/NotificationDropdown.svelte, web/src/components/notification/NotificationItem.svelte, web/src/components/Header.astro</files>
  <action>
1. Create `web/src/lib/websocket.ts`:
   - `createNotificationSocket(token: string, onMessage: (data: any) => void): { close: () => void }`:
     - Constructs WebSocket URL: `${protocol}//${location.host}/api/v1/ws/notifications?token=${token}` where protocol is ws: or wss: based on page protocol
     - Exponential backoff reconnection: start at 1s, double on each failure, max 30s, add random jitter (per research pattern)
     - On open: reset backoff to 1s
     - On message: parse JSON, call onMessage callback
     - On close: schedule reconnect with current backoff delay
     - Returns object with `close()` to stop reconnection and close socket
   - Only connect if user is authenticated (has token)

2. Create `web/src/components/notification/NotificationItem.svelte`:
   - Props: `notification` object (type, actorUsername, message, communityName, postId, isRead, createdAt)
   - Display: type prefix/icon based on notification type:
     - POST_REPLY: `[reply]` in accent
     - COMMENT_REPLY: `[reply]` in accent
     - MENTION: `[@]` in yellow/accent
   - Text: `{actorUsername} {message}` (e.g. "alice replied to your post in r/golang")
   - Timestamp: `relativeTime(createdAt)` in dim
   - Unread indicator: slightly different bg (`bg-terminal-surface/50` or left border accent) when !isRead
   - Clickable: entire row is a link to `/community/{communityName}/post/{postId}`
   - on:click handler calls markRead API and navigates

3. Create `web/src/components/notification/NotificationDropdown.svelte`:
   - State:
     - `let notifications = $state([])`, `let loading = $state(true)`
   - On mount: fetch `/api/v1/notifications?pagination.limit=20` using api() wrapper
   - Header row: "notifications" title + "mark all read" button (right-aligned)
   - List: render NotificationItem for each notification
   - Footer: "view all →" link to `/notifications`
   - "Mark all read" button: POST `/api/v1/notifications/read-all`, then update all items to isRead=true
   - When new notification arrives (passed from parent via prop or event), prepend to list
   - Empty state: "no notifications" in terminal dim
   - Terminal styling: `bg-terminal-bg border border-terminal-border rounded shadow-lg` dropdown

4. Create `web/src/components/notification/NotificationBell.svelte`:
   - State:
     - `let unreadCount = $state(0)`, `let dropdownOpen = $state(false)`, `let notifications = $state([])`
   - On mount (only if authenticated):
     - Fetch initial unread count: GET `/api/v1/notifications?unreadOnly=true&pagination.limit=1` → extract unreadCount from response
     - Start WebSocket connection via `createNotificationSocket(token, onNewNotification)`
     - onNewNotification: increment unreadCount, prepend to notifications list (if dropdown is open, it shows immediately — per user decision "silent badge updates, new notification appears at top if dropdown open")
   - Bell icon: diamond character `&#9830;` (keep existing icon)
   - Badge: small red/accent circle with count number, positioned top-right of bell. Only show when unreadCount > 0. Display "9+" for counts > 9.
   - Click handler: toggle dropdownOpen
   - Click outside: close dropdown (document click listener with cleanup)
   - Cleanup on destroy: close WebSocket connection

5. Update `web/src/components/Header.astro`:
   - Replace the static notification bell button (line 26-28) with NotificationBell Svelte component:
     ```astro
     <NotificationBell client:load />
     ```
   - Import NotificationBell in frontmatter
