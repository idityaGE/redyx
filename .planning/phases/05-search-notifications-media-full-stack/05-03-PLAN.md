Output: Fully functional notification-service with Kafka consumer, PostgreSQL storage, WebSocket hub, mention detection, and notification preferences.

<interfaces>
<!-- Proto contract for notification service -->
From proto/redyx/notification/v1/notification.proto:
- ListNotifications: pagination + unread_only filter → notifications[] + unread_count
- MarkRead: notification_id → empty
- MarkAllRead: empty → marked_count
- GetPreferences: empty → NotificationPreferences (post_replies, comment_replies, mentions, muted_communities[])
- UpdatePreferences: NotificationPreferences → NotificationPreferences
- Notification: notification_id, type (POST_REPLY/COMMENT_REPLY/MENTION), actor_id, actor_username, target_id, target_type, post_id, community_name, message, is_read, created_at

<!-- CommentEvent from Plan 01 (consumed by notification consumer) -->
From proto/redyx/events/v1/events.proto (created in Plan 01):
- CommentEvent: event_id, comment_id, post_id, author_id, author_username, parent_comment_id, parent_comment_author_id, post_author_id, community_name, body, created_at

<!-- WebSocket pattern from research -->
- nhooyr.io/websocket for Go server
- JWT token passed as query parameter on WebSocket connect
- Hub maps user_id → active connections
- Notification-service runs BOTH gRPC server (port 50059) AND HTTP server (port 8081) for WebSocket
</interfaces>

1. Create `migrations/notification/001_notifications.up.sql`:
   ```sql
   CREATE TABLE IF NOT EXISTS notifications (
     id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
     user_id UUID NOT NULL,
     type VARCHAR(30) NOT NULL,  -- 'post_reply', 'comment_reply', 'mention'
     actor_id UUID NOT NULL,
     actor_username VARCHAR(50) NOT NULL,
     target_id VARCHAR(255) NOT NULL,  -- post_id or comment_id
     target_type VARCHAR(20) NOT NULL, -- 'post' or 'comment'
     post_id VARCHAR(255) NOT NULL,    -- for navigation context
     community_name VARCHAR(50) NOT NULL,
     message TEXT NOT NULL,
     is_read BOOLEAN NOT NULL DEFAULT FALSE,
     created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
   );
   CREATE INDEX IF NOT EXISTS idx_notifications_user_unread ON notifications(user_id, is_read, created_at DESC);
   CREATE INDEX IF NOT EXISTS idx_notifications_user_created ON notifications(user_id, created_at DESC);

   CREATE TABLE IF NOT EXISTS notification_preferences (
     user_id UUID PRIMARY KEY,
     post_replies BOOLEAN NOT NULL DEFAULT TRUE,
     comment_replies BOOLEAN NOT NULL DEFAULT TRUE,
     mentions BOOLEAN NOT NULL DEFAULT TRUE,
     muted_communities TEXT[] NOT NULL DEFAULT '{}',
     updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
   );
   ```

2. Create `internal/notification/store.go`:
   - `Store` struct with `*pgxpool.Pool` and `*zap.Logger`
   - `NewStore(pool *pgxpool.Pool, logger *zap.Logger) *Store`
   - `Create(ctx, notification) (string, error)` — INSERT returning ID
   - `ListByUser(ctx, userID, unreadOnly, limit, offset) ([]Notification, int, error)` — SELECT with optional is_read filter, also return total unread count via COUNT subquery
   - `MarkRead(ctx, notificationID, userID) error` — UPDATE is_read=true WHERE id AND user_id
   - `MarkAllRead(ctx, userID) (int, error)` — UPDATE is_read=true WHERE user_id AND is_read=false, return affected count
   - `GetUnreadCount(ctx, userID) (int, error)` — COUNT WHERE user_id AND is_read=false
   - `GetPreferences(ctx, userID) (*NotificationPreferences, error)` — SELECT or return defaults if no row
   - `UpdatePreferences(ctx, userID, prefs) (*NotificationPreferences, error)` — UPSERT (INSERT ON CONFLICT UPDATE)
   - `GetUndeliveredSince(ctx, userID, since time.Time) ([]Notification, error)` — for offline notification delivery on WebSocket connect
   - Use internal Notification struct that maps to/from proto

3. Create `internal/notification/mention.go`:
   - `ExtractMentions(body string) []string` — uses `regexp.MustCompile(`(?:^|\s)u/([a-zA-Z0-9_]{3,20})`)` to find all u/username mentions in comment body
   - Returns deduplicated list of usernames (not the u/ prefix)
   - Per research, simple regex is sufficient for this pattern

4. Create `internal/notification/consumer.go`:
   - `Consumer` struct with `*kgo.Client`, `*Store`, `*Hub` (WebSocket hub), `*zap.Logger`
   - `NewConsumer(brokers []string, store *Store, hub *Hub, logger *zap.Logger) (*Consumer, error)`:
     - Consumer group: `"notification-service.redyx.comments.v1"` (unique per service, follows established pattern)
     - Topic: `"redyx.comments.v1"`
   - `Run(ctx context.Context) error`:
     - PollFetches loop
     - Deserialize CommentEvent via proto.Unmarshal
     - For each event:
       a. Check if this is a reply to a post (parent_comment_id empty → notify post_author_id with type POST_REPLY)
       b. Check if this is a reply to a comment (parent_comment_id non-empty → notify parent_comment_author_id with type COMMENT_REPLY)
       c. Extract mentions from body → for each mentioned username, resolve to user_id (needs user lookup — for v1, store by username and resolve later, OR connect to user-service DB read-only). Simplest: store actor_username as the actor, target_type="comment", and let the frontend resolve navigation.
       d. For each notification target: check preferences (is notification type enabled? is community muted?) before creating
       e. Don't notify if actor == target (don't notify yourself)
       f. Create notification in PostgreSQL store
       g. Push to WebSocket hub for real-time delivery: `hub.Send(targetUserID, notification)`
   - `Close()` method

<task type="auto">
  <name>Task 2: Create WebSocket hub, gRPC server, and service entry point</name>
  <files>internal/notification/websocket.go, internal/notification/server.go, cmd/notification/main.go</files>
  <action>
1. Create `internal/notification/websocket.go`:
   - `Hub` struct with `sync.RWMutex`, `conns map[string][]*websocket.Conn` (user_id → connections), `store *Store`, `logger *zap.Logger`
   - `NewHub(store *Store, logger *zap.Logger) *Hub`
   - `register(userID string, conn *websocket.Conn)` — adds conn to user's connection list
   - `unregister(userID string, conn *websocket.Conn)` — removes conn from list
   - `Send(userID string, notification *Notification) error` — sends JSON-encoded notification to all active connections for user via wsjson.Write. If no active connections, notification is already persisted in PostgreSQL (offline delivery handled on next connect).
   - `HandleWebSocket(w http.ResponseWriter, r *http.Request)`:
     - Extract JWT token from query parameter `?token=...` (per Pitfall 4 from research — WebSocket doesn't support custom headers after handshake)
     - Validate JWT using auth.JWTValidator
     - Accept WebSocket connection with `websocket.Accept(w, r, &websocket.AcceptOptions{OriginPatterns: []string{"*"}})`
     - Register connection
     - Deliver offline notifications: query store.GetUndeliveredSince for undelivered notifications since user's last connection, send each via wsjson.Write
     - Keep-alive read loop: read messages (client pings), break on error to trigger unregister
     - Use `nhooyr.io/websocket` (NOT gorilla/websocket — gorilla is archived per research)
   - `ServeHTTP(mux *http.ServeMux)` — registers `/api/v1/ws/notifications` handler on mux

2. Create `internal/notification/server.go`:
   - `Server` struct with `store *Store`, `hub *Hub`, `redisClient` (for unread count cache), `logger *zap.Logger`
   - `NewServer(store *Store, hub *Hub, redisClient redis.Cmdable, logger *zap.Logger) *Server`
   - Implement `ListNotifications` RPC:
     - Extract user_id from context (auth interceptor)
     - Call store.ListByUser with pagination + unread_only filter
     - Map to proto Notification messages
     - Include unread_count in response
   - Implement `MarkRead` RPC:
     - Extract user_id, validate notification_id
     - Call store.MarkRead
     - Invalidate Redis unread count cache for user
   - Implement `MarkAllRead` RPC:
     - Extract user_id
     - Call store.MarkAllRead
     - Invalidate Redis unread count cache
     - Return marked_count
   - Implement `GetPreferences` RPC:
     - Extract user_id
     - Call store.GetPreferences (returns defaults if no row)
     - Map to proto NotificationPreferences
   - Implement `UpdatePreferences` RPC:
     - Extract user_id, validate request
     - Call store.UpdatePreferences (UPSERT)
     - Return updated preferences
   - Mark ListNotifications, MarkRead, MarkAllRead, GetPreferences, UpdatePreferences as auth-required (not public)

3. Create `cmd/notification/main.go`:
   - Follow pattern from `cmd/comment/main.go`
   - Load config for "notification" service
   - Connect to PostgreSQL (notifications database)
   - Run migrations from `migrations/notification/`
   - Connect to Redis DB 8 (notification-service's reserved DB)
   - Create JWT validator
   - Create Store, Hub, Server
   - Create gRPC server with middleware chain on port 50059
   - Register NotificationServiceServer
   - Start Kafka Consumer in goroutine
   - Start HTTP server on WebSocketPort (8081) for WebSocket endpoint in a separate goroutine:
     ```go
     mux := http.NewServeMux()
     hub.ServeHTTP(mux)
     go func() {
       addr := fmt.Sprintf(":%d", cfg.WebSocketPort)
       logger.Info("websocket server starting", zap.String("addr", addr))
       if err := http.ListenAndServe(addr, mux); err != nil {
         logger.Fatal("websocket server failed", zap.Error(err))
       }
     }()
     ```
   - Block on gRPC srv.Run()
