1. Create `web/src/components/search/SearchBar.svelte`:
   - Props: none (self-contained, reads route from window.location)
   - State (Svelte 5 runes):
     - `let query = $state('')`
     - `let suggestions = $state([])` — community autocomplete results
     - `let showDropdown = $state(false)`
     - `let communityScope = $state(null)` — auto-detected or manually cleared
   - Auto-detect community scope on mount:
     ```typescript
     let communityScope = $derived(
       typeof window !== 'undefined'
         ? window.location.pathname.match(/^\/community\/([^/]+)/)?.[1] ?? null
         : null
     );
     ```
     Store in a writable $state so user can clear it (click X on pill).
   - Debounced autocomplete: use `$effect` with a timer pattern (300ms debounce). When query.length >= 2, fetch `/api/v1/search/communities?query={query}&limit=8` using the api() wrapper. Show results in dropdown.
   - Dropdown: positioned absolutely below the search input. Each suggestion shows community name + member count. Clicking a suggestion navigates to `/community/{name}`. Terminal-style: `bg-terminal-bg border border-terminal-border` with `text-terminal-fg` items.
   - Community scope pill: when communityScope is set, show a removable tag/pill next to the `>` prefix: `[r/{name} ×]` in green accent. Clicking × clears the scope.
   - On Enter: navigate to `/search?q={query}` (append `&community={communityScope}` if scoped). Use `window.location.href` or Astro navigation.
   - On Escape: close dropdown, clear suggestions.
   - Click outside: close dropdown (use document click listener with cleanup in $effect).
   - Keep the `>` prefix in terminal style per existing Header pattern.
