  <name>Task 1: Create event proto definitions and add config for new services</name>
  <files>proto/redyx/events/v1/events.proto, internal/platform/config/config.go</files>
  <action>
1. Create `proto/redyx/events/v1/events.proto` with:
   - `CommentEvent` message: event_id, comment_id, post_id, author_id, author_username, parent_comment_id, parent_comment_author_id, post_author_id, community_name, body, created_at (google.protobuf.Timestamp). This is the data notification-service needs to generate reply notifications and detect u/username mentions.
   - `PostEvent` message: event_id, post_id, title, body, author_username, community_name, vote_score (int32), comment_count (int32), created_at (google.protobuf.Timestamp), event_type (enum: CREATED, UPDATED, DELETED). This is what search-service indexes.
   - Package: `redyx.events.v1`, go_package: `github.com/idityaGE/redyx/gen/redyx/events/v1;eventsv1`

2. Run `make proto` to generate Go code + rebuild Envoy descriptor.

3. Add new config fields to `internal/platform/config/config.go`:
   - `MeilisearchURL` (string, default "http://meilisearch:7700")
   - `MeilisearchAPIKey` (string, default "dev-master-key")
   - `MinIOEndpoint` (string, default "http://minio:9000")
   - `MinIOAccessKey` (string, default "minioadmin")
   - `MinIOSecretKey` (string, default "minioadmin")
   - `MinIOBucket` (string, default "redyx-media")
   - `WebSocketPort` (int, default 8081)
   Follow the existing pattern with envStr/envInt helpers.

4. Install new Go dependencies:

   ```bash
   go get github.com/aws/aws-sdk-go-v2/credentials@latest
   ```

<task type="auto">
  <name>Task 2: Add Kafka producers to comment-service and post-service</name>
  <files>internal/comment/producer.go, internal/post/producer.go, internal/comment/server.go, internal/post/server.go, cmd/comment/main.go, cmd/post/main.go</files>
  <action>
1. Create `internal/comment/producer.go`:
   - Follow the exact pattern from `internal/vote/kafka.go`
   - Topic constant: `TopicComments = "redyx.comments.v1"`
   - `CommentProducer` struct wrapping `*kgo.Client`
   - `NewCommentProducer(brokers []string) (*CommentProducer, error)` — creates kgo.Client with topic
   - `Publish(ctx context.Context, event *eventsv1.CommentEvent) error` — proto.Marshal + kgo.Produce with `context.Background()` (fire-and-forget, same as vote-service pattern per [03-07] decision)
   - `Close()` method

2. Create `internal/post/producer.go`:
   - Same pattern as comment producer
   - Topic constant: `TopicPosts = "redyx.posts.v1"`
   - `PostProducer` struct
   - `NewPostProducer(brokers []string) (*PostProducer, error)`
   - `Publish(ctx context.Context, event *eventsv1.PostEvent) error`
   - `Close()` method

3. Modify `internal/comment/server.go`:
   - Add `producer *CommentProducer` field to Server struct
   - Update `NewServer` to accept producer parameter
   - In `CreateComment` RPC, after successful ScyllaDB insert, publish CommentEvent with all fields populated. Use `context.Background()` for fire-and-forget produce (not request context, per [03-07] decision). Generate event_id with `uuid.New().String()`.
   - Need to pass post_author_id into the event — the comment store should have access to this from the parent post lookup that already happens for reply validation.

4. Modify `internal/post/server.go`:
   - Add `producer *PostProducer` field to Server struct
   - Update `NewServer` to accept producer parameter
   - In `CreatePost` RPC, after successful insert, publish PostEvent with event_type=CREATED
   - In `UpdatePost` RPC, after successful update, publish PostEvent with event_type=UPDATED
   - In `DeletePost` RPC, after successful delete, publish PostEvent with event_type=DELETED
   - Use `context.Background()` for all publishes

5. Update `cmd/comment/main.go`:
   - Create CommentProducer from KafkaBrokers config
   - Pass producer to comment.NewServer
   - Defer producer.Close()

6. Update `cmd/post/main.go`:
   - Create PostProducer from KafkaBrokers config
   - Pass producer to post.NewServer (or however the server is constructed)
   - Defer producer.Close()
  </action>
  <verify>
    <automated>go build ./cmd/comment/... && go build ./cmd/post/...</automated>
  </verify>
  <done>Both services compile with Kafka producers, CommentEvent published on comment creation, PostEvent published on post create/update/delete</done>
</task>
