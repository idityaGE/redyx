Dropdown styling: `absolute right-0 top-full mt-1 bg-terminal-bg border border-terminal-border text-xs font-mono z-20` with each item as a clickable row.

Close dropdown on click outside (use onclick handler on window or a backdrop div).

  <name>Task 2: Ban banner, pinned post display, and form disabling</name>
  <files>web/src/components/community/CommunityDetail.svelte, web/src/components/post/FeedList.svelte, web/src/components/post/FeedRow.svelte</files>

**1. Add ban banner to `CommunityDetail.svelte`:**
On mount, if user is authenticated, check ban status:
```javascript
const banCheck = await api(`/communities/${name}/moderation/check-ban`, {
  method: 'POST',
  body: JSON.stringify({ communityName: name, userId: currentUser.userId })
});
```
If `banCheck.isBanned`:
- Show a prominent banner at the top of the community page, below the header:
  ```
  ┌─ banned ─────────────────────────────────────────┐
  │ You are banned from this community.              │
  │ Reason: {reason}                                  │
  │ Expires: {formattedDate or "Permanent"}           │
  └──────────────────────────────────────────────────┘
  ```
  Styled: `border border-red-500/50 bg-red-500/10 text-red-400 font-mono text-sm p-3 mb-4`
- Set a `isBanned` state to true
- Pass `isBanned` down to child components (post submit form, etc.) to disable creation

If ban check fails (service unavailable), silently continue (fail-open).

**2. Disable post/comment forms for banned users:**
Pass `isBanned` prop from CommunityDetail → post creation components and comment forms.
When `isBanned=true`:
- Post submit button disabled, shows `[banned — cannot post]`
- Comment form textarea disabled, placeholder "You are banned from this community"

**3. Pinned post display in `FeedList.svelte`:**
When rendering the community feed (not home feed), check for pinned posts:
- If any post in the data has `isPinned: true`, render them at the TOP of the list, BEFORE regular posts
- Pin icon/label: Add `📌` or `[pinned]` text label next to the post title for pinned posts
- Styling: Pinned posts get a subtle left border accent: `border-l-2 border-accent-500` to distinguish

In `FeedRow.svelte`:
- If `post.isPinned`, show `[pinned]` tag in accent color next to the title
- Slightly different background: `bg-terminal-surface/50` to make them stand out

The server should return pinned posts with `isPinned: true` field — this field already exists in the post proto and is scanned in queries (per STATE.md). The frontend just needs to sort pinned posts to the top.
