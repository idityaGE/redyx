<task type="auto">
  <name>Task 1: Docker service startup and API curl verification</name>
  <files></files>
  <action>
**1. Start all services:**
```bash
docker compose up -d --build
```
Wait for all services to be healthy. Check:
```bash
docker compose ps
```
Verify moderation-service and spam-service are running.

**2. API verification sequence using curl through Envoy (localhost:8080):**

Prerequisites: Register a test user and create a test community, or use existing ones from prior phases. Get a JWT token by logging in.

**Spam checks (SPAM-01, SPAM-02, SPAM-03):**
a. Create post with blocked keyword — expect 400 with vague rejection:
```bash
curl -X POST http://localhost:8080/api/v1/communities/{community}/posts \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"title":"buy now free money","body":"click here","communityName":"{community}"}'
```
Expected: 400 or equivalent error, vague message (NOT revealing matched keyword)

b. Create same post twice — expect duplicate rejection on second attempt:
```bash
curl -X POST http://localhost:8080/api/v1/communities/{community}/posts \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"title":"Unique test post","body":"Test body content for dedup","communityName":"{community}"}'
# Second identical request:
curl -X POST http://localhost:8080/api/v1/communities/{community}/posts \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"title":"Unique test post","body":"Test body content for dedup","communityName":"{community}"}'
```
Expected: First succeeds, second fails with duplicate error

**Moderation actions (MOD-01 through MOD-06):**
c. Submit a report (as regular user):
```bash
curl -X POST http://localhost:8080/api/v1/communities/{community}/moderation/reports \
  -H "Authorization: Bearer $USER_TOKEN" \
  -d '{"contentId":"{post_id}","contentType":1,"reason":"Spam"}'
```
Expected: 200 with report_id

d. List report queue (as moderator):
```bash
curl http://localhost:8080/api/v1/communities/{community}/moderation/reports?status=active \
  -H "Authorization: Bearer $MOD_TOKEN"
```
Expected: 200 with reports array containing the submitted report

e. Ban a user (as moderator):
```bash
curl -X POST http://localhost:8080/api/v1/communities/{community}/moderation/ban \
  -H "Authorization: Bearer $MOD_TOKEN" \
  -d '{"userId":"{user_id}","username":"testuser","reason":"Spam","durationSeconds":86400}'
```
Expected: 200

f. Check ban status:
```bash
curl -X POST http://localhost:8080/api/v1/communities/{community}/moderation/check-ban \
  -H "Authorization: Bearer $MOD_TOKEN" \
  -d '{"communityName":"{community}","userId":"{user_id}"}'
```
Expected: 200 with is_banned=true

g. Try creating a post as banned user:
```bash
curl -X POST http://localhost:8080/api/v1/communities/{community}/posts \
  -H "Authorization: Bearer $BANNED_USER_TOKEN" \
  -d '{"title":"Should fail","body":"Banned user post","communityName":"{community}"}'
```
Expected: 403 Forbidden

h. Pin a post (as moderator):
```bash
curl -X POST http://localhost:8080/api/v1/communities/{community}/moderation/pin \
  -H "Authorization: Bearer $MOD_TOKEN" \
  -d '{"postId":"{post_id}","communityName":"{community}"}'
```
Expected: 200

i. View mod log:
```bash
curl http://localhost:8080/api/v1/communities/{community}/moderation/log \
  -H "Authorization: Bearer $MOD_TOKEN"
```
Expected: 200 with entries for ban and pin actions

j. List bans:
```bash
curl http://localhost:8080/api/v1/communities/{community}/moderation/bans \
  -H "Authorization: Bearer $MOD_TOKEN"
```
Expected: 200 with bans array containing the banned user

k. Unban user:
```bash
curl -X POST http://localhost:8080/api/v1/communities/{community}/moderation/unban \
  -H "Authorization: Bearer $MOD_TOKEN" \
  -d '{"userId":"{user_id}","communityName":"{community}"}'
```
Expected: 200

**3. Fix any issues found during curl testing.**
Log all issues, fix root causes, rebuild services, re-test. Common issues to watch for:
- Envoy routing (moderation regex BEFORE community catch-all)
- Proto descriptor missing services
- Auth interceptor blocking service-to-service calls
- Redis connection errors (wrong DB number)
- Missing environment variables in docker-compose
