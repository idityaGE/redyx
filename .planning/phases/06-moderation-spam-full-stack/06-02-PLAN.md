  <files>internal/spam/blocklist.go, internal/spam/dedup.go, internal/spam/server.go, internal/spam/data/blocklist.json</files>
  <behavior>
    - CheckContent with clean content returns CLEAN result, no reasons
    - CheckContent with blocked keyword returns SPAM result with "keyword_match" reason
    - CheckContent with blocked URL domain returns SPAM result with "url_blocklist" reason
    - CheckContent with duplicate content hash returns CLEAN result but is_duplicate=true
    - CheckContent with first-time content stores hash in Redis and returns is_duplicate=false
    - Content hash is SHA-256 of normalized text (lowercase, trimmed, collapsed whitespace)
    - Blocklist matching is case-insensitive
    - URL extraction handles markdown links [text](url) and bare URLs
  </behavior>
  <action>
**1. Create `internal/spam/data/blocklist.json`:**
```json
{
  "keywords": [
    "buy now", "click here", "free money", "act now", "limited time offer",
    "congratulations you won", "nigger", "make money fast", "work from home",
    "casino online", "viagra", "crypto guaranteed"
  ],
  "domains": [
    "bit.ly", "tinyurl.com", "malware-site.com", "phishing-example.com",
    "spam-domain.xyz", "free-money.biz", "casino-bonus.net"
  ]
}
```

**2. Create `internal/spam/blocklist.go`:**
- `type Blocklist struct { keywords []string; domains map[string]bool }` 
- `func LoadBlocklist(path string) (*Blocklist, error)` — reads JSON file, normalizes keywords to lowercase
- `func (b *Blocklist) CheckKeywords(content string) (bool, string)` — case-insensitive substring match against keyword list, returns (matched, keyword)
- `func (b *Blocklist) CheckURLs(urls []string) (bool, string)` — parse each URL with `net/url`, extract hostname, check against domains map. Returns (matched, domain)
- URL extraction helper: `func ExtractURLs(content string) []string` — regex for `https?://\S+` and markdown `\[.*?\]\((https?://\S+?)\)` patterns

**3. Create `internal/spam/dedup.go`:**
- `type DedupChecker struct { rdb *redis.Client }` + `func NewDedupChecker(rdb) *DedupChecker`
- `func (d *DedupChecker) Check(ctx, userID, content string) (hash string, isDuplicate bool, err error)` 
- Normalize content: `strings.ToLower(strings.TrimSpace(content))`, collapse multiple whitespace to single space
- SHA-256 hash of normalized content
- Redis key: `dedup:{userID}:{hash}`, SET NX with 24h TTL
- If SET returns false (key already exists) → isDuplicate = true
- Returns the hash string in all cases (for CheckContentResponse.content_hash)

**4. Create `internal/spam/server.go`:**
- `type Server struct { blocklist *Blocklist; dedup *DedupChecker; logger *zap.Logger }`
- `func NewServer(blocklist, dedup, logger) *Server`
- `CheckContent` implementation:
  1. Check keywords in content → if match, return SPAM with reason "blocked_content"
  2. Extract URLs from content field, combine with request urls field
  3. Check URLs against domain blocklist → if match, return SPAM with reason "blocked_url"
  4. Check duplicate via dedup.Check → set is_duplicate and content_hash on response
  5. If no matches, return CLEAN
  - IMPORTANT: Return vague reasons only — do NOT include the specific keyword or URL that matched
- `ReportSpam` implementation: Forward to moderation service SubmitReport with source="spam-detection". For now, log and return a generated report_id (UUID). Integration with moderation service happens via Kafka consumer (SPAM-04).

**1. Create `internal/spam/consumer.go`:**
Follow `internal/vote/consumer.go` pattern exactly:
- `type BehaviorConsumer struct { client *kgo.Client; rdb *redis.Client; moderationClient modv1.ModerationServiceClient; logger *zap.Logger }`
- `func NewBehaviorConsumer(kafkaBrokers []string, rdb, moderationClient, logger) (*BehaviorConsumer, error)`
  - Consumer group: `spam-service.redyx.posts.v1`
  - Topic: `redyx.posts.v1` (reuse existing post events topic)
  - Also create behavior-specific topic `redyx.behavior.v1` with `kadm.CreateTopics` for future extensibility
- `func (c *BehaviorConsumer) Run(ctx context.Context)` — PollFetches loop
- `func (c *BehaviorConsumer) analyzePostBehavior(ctx, event *eventsv1.PostEvent)`:
  - Only process POST_EVENT_TYPE_CREATED events
  - **Rapid posting detection:** Redis sliding window `behavior:rapid:{authorUsername}` — INCR + EXPIRE 5min. If count > 10 posts in 5 minutes → flag.
  - **Link spam detection:** Redis sliding window `behavior:links:{authorUsername}` — count posts containing URLs in last hour. If > 5 link posts in 1 hour → flag.
  - **Flagging:** When threshold exceeded, call moderation service SubmitReport via gRPC with source="spam-detection", reason="rapid_posting" or "link_spam", content_id=post_id, community_name from event.

  Thresholds (Claude's discretion per CONTEXT.md):
  - Rapid posting: >10 posts in 5 minutes
  - Link spam: >5 posts with URLs in 1 hour
