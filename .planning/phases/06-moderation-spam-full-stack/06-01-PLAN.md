    - RemoveContent: Sets is_deleted on target post/comment via cross-service gRPC, logs to mod_log, returns error if not moderator
    - BanUser: Creates ban record with expiry, optionally removes all user content, logs to mod_log
    - UnbanUser: Deletes ban record, logs to mod_log
    - PinPost: Sets is_pinned on post, rejects if community already has 2 pins, logs to mod_log
    - UnpinPost: Clears is_pinned on post, logs to mod_log
    - GetModLog: Returns paginated mod_log entries, filterable by action type
    - ListReportQueue: Returns aggregated reports grouped by content_id, filterable by status (active/resolved) and source
    - SubmitReport: Creates report record, requires authenticated user
    - DismissReport: Marks report as resolved with dismissed action, logs to mod_log
    - RestoreContent: Re-shows removed content (unsets is_deleted) via cross-service gRPC, marks related reports as resolved, logs to mod_log
    - ListBans: Returns active bans for community (expired auto-filtered via WHERE clause)
    - CheckBan: Returns ban status for user in community (uses Redis cache with fallback to DB)
  </behavior>

**1. Create migration `migrations/moderation/001_moderation.up.sql`:**
Use the schema from 06-RESEARCH.md code examples section (reports, bans, mod_log tables with indexes). This is the exact schema — copy it.

**2. Create `internal/moderation/store.go`:**
PostgreSQL store following `internal/notification/store.go` pattern:
- `type Store struct { pool *pgxpool.Pool }` + `func NewStore(pool) *Store`
- Methods: `CreateReport`, `GetReportsByContent`, `ListReports(communityID, status, source, pagination)`, `UpdateReportStatus(contentID, contentType, action, resolvedBy)`, `CreateBan`, `DeleteBan(communityID, userID)`, `GetBan(communityID, userID)`, `ListBans(communityID, pagination)`, `CreateModLogEntry`, `ListModLog(communityID, actionFilter, pagination)`
- Reports query uses `GROUP BY content_id, content_type` with `COUNT(*)` as report_count and `MAX(created_at)` for latest report, sorted by report_count DESC
- Bans query filters `WHERE expires_at IS NULL OR expires_at > now()` for active bans only
- Use `pgxpool.Pool` and `pgx.CollectRows` patterns consistent with other stores

**3. Create `internal/moderation/server.go`:**
- `type Server struct` with: store, logger, communityClient, postClient, commentClient, redisClient
- `verifyModerator(ctx, communityName)` helper: extract claims from context, call communityClient.GetCommunity with userId, check isModerator field. Return error if not moderator. Pattern from research.
- Each mod RPC: verify moderator → execute action → log to mod_log → return
- `RemoveContent`: Call postClient or commentClient to set is_deleted based on content_type. This requires adding internal RPCs to post-service and comment-service (handled in Plan 03).
  For now, implement the moderation server method that calls `s.postClient.ModeratorRemovePost()` / `s.commentClient.ModeratorRemoveComment()` — these will be stubbed as UNIMPLEMENTED until Plan 03.
- `BanUser`: Create ban in store, cache in Redis (key: `ban:{communityID}:{userID}`, value: JSON with reason+expires, TTL matching ban duration or no expiry for permanent). If `remove_content` is true, call post-service and comment-service to remove all content by user in community.
- `PinPost`: Count existing pins for community via postClient, reject if >= 2. Then call postClient to set is_pinned. The pin count check needs a new internal RPC on post-service (Plan 03).
- `CheckBan`: Check Redis first (`ban:{communityID}:{userID}`), fallback to DB store.GetBan.
- `SubmitReport`: Verify authenticated (claims != nil), create report with source="user".
- `DismissReport`: Verify moderator, update report status, log to mod_log.
- `RestoreContent`: Verify moderator, call post/comment service to undelete, update related reports to resolved, log to mod_log.
