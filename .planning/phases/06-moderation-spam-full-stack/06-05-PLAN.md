**1. Create `web/src/components/moderation/ReportQueue.svelte`:**
Props: `{ communityName: string }`

Layout:
- Tab bar at top: `[active]` `[resolved]` — toggle between active and resolved reports
- Source filter: `[all] [user-report] [spam-detection]` — reuse SortBar pattern
- List of report items, each showing:
  - Source tag: `[user-report]` or `[spam-detection]` with different colors (dim vs accent)
  - Content type: `[post]` or `[comment]`
  - Content title/preview (truncated to 80 chars)
  - Content author username
  - Report count: `{count} report(s)`
  - Report reason (most common)
  - Timestamp of latest report

Active tab — inline action buttons per item (locked decision):
  - `[remove]` `[dismiss]` `[ban user]` — each with inline confirmation pattern (confirmingX state → [confirm?]/[cancel])
  - [remove] calls POST `/moderation/remove` with contentId + contentType
  - [dismiss] calls POST `/moderation/reports/dismiss` with contentId + contentType
  - [ban user] opens BanDialog with userId + username from report

Resolved tab — each item shows:
  - Same content info as active
  - Resolved action taken: "removed" / "dismissed" / "banned"
  - `[undo]` button → calls POST `/moderation/restore` for removes, or `/moderation/unban` for bans, or re-opens report for dismissals. Moves item back to active queue.

Pagination: Use cursor-based pagination pattern from FeedList.

Fetch: GET `/api/v1/communities/${communityName}/moderation/reports?status=${activeTab}&source=${sourceFilter}`
Refresh list after any action.
