**1. Create `internal/post/moderator.go`:**
Implement the new internal RPCs on the existing post Server struct:
- `ModeratorRemovePost`: Use shard router to find correct shard, execute `UPDATE posts SET is_deleted = true WHERE id = $1 AND community_name = $2`
- `ModeratorRestorePost`: `UPDATE posts SET is_deleted = false WHERE id = $1 AND community_name = $2`
- `SetPostPinned`: `UPDATE posts SET is_pinned = $1 WHERE id = $2 AND community_name = $3`
- `CountPinnedPosts`: Use shard router, `SELECT COUNT(*) FROM posts WHERE community_name = $1 AND is_pinned = true AND is_deleted = false`
- `RemovePostsByUser`: `UPDATE posts SET is_deleted = true WHERE author_id = $1 AND community_name = $2`, return count of affected rows

For shard routing: use the same `s.shardRouter.GetShard(communityName)` that CreatePost uses. All posts for a community are on the same shard.

**2. Create `internal/comment/moderator.go`:**
Implement internal RPCs on existing comment Server struct:
- `ModeratorRemoveComment`: Update both ScyllaDB tables (comments_by_post and comments_by_id) to set is_deleted = true. Use comment_id + post_id as the key.
- `ModeratorRestoreComment`: Set is_deleted = false on both tables.
- `RemoveCommentsByUser`: Query comments_by_id WHERE author_id = user_id, then update each matching comment. For v1, this may require a full table scan — acceptable for moderation actions which are infrequent. Alternatively, use an ALLOW FILTERING query on author_id if an index exists, or iterate by post.

**3. Modify `internal/post/server.go`:**
- Add `spamClient spamv1.SpamServiceClient` and `moderationClient modv1.ModerationServiceClient` fields to Server struct
- In `CreatePost`, BEFORE persisting to DB, add:
  1. Ban check: `s.moderationClient.CheckBan(ctx, &modv1.CheckBanRequest{CommunityName: communityName, UserId: claims.UserID})` — if is_banned, return `perrors.ErrForbidden` with message "You are banned from this community"
  2. Spam check: `s.spamClient.CheckContent(ctx, &spamv1.CheckContentRequest{UserId: claims.UserID, ContentType: "post", Content: title + "\n" + body, Urls: extractedUrls})` — if result == SPAM, return `perrors.ErrInvalidInput` with vague message "Your post couldn't be published — it may contain restricted content." If is_duplicate, return `perrors.ErrAlreadyExists` with "Duplicate content detected."
  3. Fail-open: If spam/ban check returns a gRPC error (service unavailable), log warning and allow content through. This matches the project's fail-open rate limiting pattern.

**4. Modify `internal/comment/server.go`:**
- Add `spamClient spamv1.SpamServiceClient` and `moderationClient modv1.ModerationServiceClient` fields
- In `CreateComment`, add same ban check and spam check pattern as posts
- Fail-open on service errors

**5. Modify `cmd/post/main.go` and `cmd/comment/main.go`:**
- Add spam-service and moderation-service gRPC client connections
- New env vars: SPAM_SERVICE_ADDR, MODERATION_SERVICE_ADDR
- Pass clients to Server struct constructor
