    **1. Create VoteEvent proto** at `proto/redyx/common/v1/events.proto`:
    ```protobuf
    syntax = "proto3";
    package redyx.common.v1;
    import "google/protobuf/timestamp.proto";
    
    message VoteEvent {
      string event_id = 1;
      string user_id = 2;
      string target_id = 3;
      string target_type = 4;  // "post" or "comment"
      string author_id = 5;
      int32 score_delta = 6;
      string community_id = 7;
      google.protobuf.Timestamp occurred_at = 8;
    }
    ```
    Run `make proto` to regenerate Go code + descriptor.

    **2. Extend pagination/cursor.go** — add `EncodeSortCursor` and `DecodeSortCursor` that encode `(sort_value_float64, id, created_at)` for cross-shard keyset pagination on hot_score and vote_score. Keep the existing EncodeCursor/DecodeCursor unchanged.

    **3. Add config fields** to `internal/platform/config/config.go`:
    - `KafkaBrokers string` — loaded from `KAFKA_BROKERS` env var, default `"kafka:9092"`
    - `PostShardDSNs []string` — loaded from `POST_SHARD_DSNS` env var (comma-separated), default `"postgres://redyx:dev@postgres:5432/posts_shard_0?sslmode=disable,postgres://redyx:dev@postgres:5432/posts_shard_1?sslmode=disable"`

    **4. Create shard router** at `internal/post/shard.go`:
    - Add `github.com/serialx/hashring` dependency: `go get github.com/serialx/hashring`
    - `ShardRouter` struct with `ring *hashring.HashRing`, `pools map[string]*pgxpool.Pool`, `shardIDs []string`
    - `NewShardRouter(dsns []string) (*ShardRouter, error)` — creates a pgxpool.Pool per DSN, names them `shard_0`, `shard_1`, builds hash ring with 40 virtual nodes per shard
    - `GetPool(communityID string) (*pgxpool.Pool, string)` — returns pool + shard name for a community
    - `AllPools() []*pgxpool.Pool` — returns all pools for cross-shard queries
    - `Close()` — closes all pools

    **5. Create ranking functions** at `internal/post/ranking.go`:
    - `HotScore(score int, createdAt time.Time) float64` — Lemmy algorithm: `10000 * log(max(1, 3+score)) / (hoursAge+2)^1.8`
    - `RisingScore(score int, createdAt time.Time) float64` — velocity: `score / max(1, hoursAge)`

    **6. Update init-databases.sql** — add `posts_shard_0` and `posts_shard_1` databases with GRANT ALL to redyx.

    **7. Create migration files** (identical for both shards):
    - `migrations/post_shard_0/001_create_posts.up.sql` and `migrations/post_shard_1/001_create_posts.up.sql`
    - Schema from RESEARCH.md: posts table with all columns (id, title, body, url, post_type, author_id, author_username, community_id, community_name, vote_score, upvotes, downvotes, comment_count, hot_score, is_edited, is_deleted, is_pinned, is_anonymous, thumbnail_url, created_at, edited_at)
    - All indexes: idx_posts_community_hot, idx_posts_community_new, idx_posts_community_top, idx_posts_community_id, idx_posts_author
    - saved_posts table with (user_id, post_id, saved_at) PK and idx_saved_posts_user index

  <name>Task 2: PostService gRPC server — CRUD + community feed + anonymous posts + saved posts</name>
  <files>

  <action>
    **1. Create Redis cache** at `internal/post/cache.go`:
    - Cache layer for community membership lookups (from community-service gRPC call, cached 5min)
    - Feed cache: `feed:{user_id}:{sort}:{time_range}:{cursor_hash}` with 2min TTL
    - Score cache read: `votes:score:{post_id}` from Redis DB 5 (vote-service's Redis — read-only)
    - Use Redis DB 4 for post-service cache.

    **2. Create gRPC server** at `internal/post/server.go`:
    Follow the `internal/community/server.go` pattern exactly.

    Server struct:
    ```go
    type Server struct {
        postv1.UnimplementedPostServiceServer
        shards *ShardRouter
        cache  *Cache
        logger *zap.Logger
    }
    ```

    **CreatePost RPC (POST-01, POST-02, POST-03, POST-08)**:
    - Require auth (claims must be non-nil)
    - Validate: title 1-300 chars, body max 40K (text), URL valid (link), media type returns UNIMPLEMENTED for now (POST-03 stub)
    - Resolve community: call community-service via gRPC or use cached community data (name → id mapping). For v1 simplicity, make a direct DB query or HTTP call to the community API. Use cache layer.
    - Route to correct shard: `shards.GetPool(communityID)`
    - Insert post with computed initial hot_score (HotScore(0, now))
    - If `is_anonymous`, store real author_id in DB but return `author_username = "[anonymous]"` and `author_id = ""` in the response (POST-08, Pitfall 7)
    - Return CreatePostResponse with full Post

    **GetPost RPC (POST-04)**:
    - Accept post_id, need to find which shard — query all shards in parallel (post_id is UUID, not shard-routable). Use AllPools().
    - If `is_anonymous` and requester is NOT a moderator of the community, mask author_id and author_username
    - Include `user_vote` field — read from Redis `votes:state:{user_id}:{post_id}` (vote-service Redis DB 5). If no auth, user_vote = 0.
    - Include `is_saved` field — check saved_posts table for current user.
    - Return GetPostResponse

    **UpdatePost RPC (POST-05)**:
    - Require auth, post must exist, caller must be author (author_id matches claims.UserID)
    - Update title/body/url, set is_edited=true, edited_at=now()
    - Invalidate any cached feed entries for the community

    **DeletePost RPC (POST-05)**:
    - Require auth, caller must be author OR community moderator
    - Soft delete: SET is_deleted=true, title="[deleted]", body="[deleted]"

    **ListPosts RPC (POST-06)**:
    - Single-shard query: resolve community_name → community_id → shard
    - Support all sort orders:
      - HOT: ORDER BY hot_score DESC, id DESC
      - NEW: ORDER BY created_at DESC, id DESC
      - TOP: ORDER BY vote_score DESC, id DESC — with time_range filter (WHERE created_at > now() - interval)
      - RISING: Compute rising score on-the-fly for recent posts (last 24h), ORDER BY computed value
    - WHERE is_deleted = false
    - Cursor-based pagination using sort-aware cursor
    - Return ListPostsResponse with posts + pagination

    **ListHomeFeed RPC (POST-07)**:
    - Require auth
    - Get user's joined community IDs (call community-service ListCommunities filtered by user, or cache)
    - Group community IDs by shard
    - Query each shard in parallel: SELECT FROM posts WHERE community_id IN (...) with sort/time_range
    - Merge-sort results from all shards
    - Apply cursor pagination to merged results
    - Cache result page in Redis (2min TTL)
    - Return ListHomeFeedResponse

    **SavePost RPC (POST-09)**:
    - Require auth
    - If save=true: INSERT INTO saved_posts ON CONFLICT DO NOTHING
    - If save=false: DELETE FROM saved_posts WHERE user_id AND post_id
    - Post can be on any shard — saved_posts table is on shard_0 (centralized)

    **ListSavedPosts RPC (POST-09)**:
    - Require auth
    - Query saved_posts for user_id, ordered by saved_at DESC
    - For each saved post_id, fetch actual post data from the appropriate shard
    - Batch by shard for efficiency
    - Return ListSavedPostsResponse with posts + pagination
  </action>

    8. Register PostServiceServer
    9. Start background goroutine for periodic hot_score refresh:

       - `UPDATE posts SET hot_score = [HotScore formula in SQL] WHERE created_at > now() - interval '48 hours' AND is_deleted = false`
       - The SQL formula: `10000.0 * ln(GREATEST(1, 3 + vote_score)) / POWER(EXTRACT(EPOCH FROM (now() - created_at)) / 3600.0 + 2, 1.8)`
       - Log execution time and rows affected
    10. Run server (blocks until shutdown)
    11. On shutdown: cancel hot score goroutine, close shard pools, close Redis
